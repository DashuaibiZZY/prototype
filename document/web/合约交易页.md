# 1\. 导航栏

导航栏整体功能（菜单、语言切换、链信息、用户钱包等）详见《[需求文档 \- 统一顶部导航栏](https://ksgf73ukjokp.sg.larksuite.com/docx/J7Y5d6w9moPT6JxQIgalbqybgif)》，本文档仅补充合约交易页特有的功能。



## 1\.1 偏好设置抽屉

**1\.1\.1 触发方式**

- 点击导航栏右侧（设置）图标，从页面右侧滑入偏好设置抽屉面板。

- 点击抽屉外部的半透明遮罩层，或点击抽屉右上角关闭按钮，面板滑出关闭。

- 打开时页面主体不可滚动，关闭后恢复。

    

**1\.1\.2 抽屉内容**

**（1）下单确认**

该区域包含三个开关项，控制不同类型订单的下单二次确认弹窗是否弹出：

- 限价订单：开关默认开启。

- 市价订单：开关默认开启。

- 止盈止损订单：开关默认开启。



每个开关项为 toggle 开关样式。用户可单独设置每类订单是否需要二次确认。



**（2）消息通知**

- 成交提示音：开关默认开启，控制下单成功/失败等事件是否播放提示音。



**（3）底部操作**

- “恢复至默认设置”按钮：点击将所有偏好设置项恢复为系统默认值。

    

**1\.1\.3 数据持久化**

- 用户的偏好设置需保存至后端或本地存储，下次登录时保持用户上次的设置状态。

# 2\. 市场信息栏

## 2\.1 功能概述

展示当前交易对的核心市场数据，支持快速切换交易对。



## 2\.2 当前交易对基础信息展示



**2\.2\.1 交易对名称与杠杆**

- 左侧显示当前交易对（如 BTCUSDT），字体加粗。

- 交易对名称右侧附带该交易对支持的最大杠杆倍数标签（如“100x”）。

- 点击该区域弹出交易对切换浮窗（见 2\.3）。

    

**2\.2\.2 最新价与涨跌幅**

- 最新成交价大字体显示，24小时涨跌幅小字显示在价格下方。

- 颜色规则：涨（≥0）为绿色，跌（\<0）为红色。

    

**2\.2\.3 核心市场指标**

以多列并排展示以下数据项：

|数据项|说明|交互|
|---|---|---|
|标记价格|根据现货指数和资金费率计算的合理价格，用于计算未实现盈亏和强平|- 带虚线下划线，hover 显示解释 tooltip<br>- 文案：标记价格用于计算您的强平价格和持仓保证金。标记价格源自经过处理的指数价格。|
|指数价格|现货指数价格|- 带虚线下划线，hover 显示解释 tooltip<br>- 文案：指数价格是基于多家主流交易所价格进行加权平均计算得出的综合价格。|
|资金费率 / 倒计时|当前资金费率及距离下次结算的倒计时（格式：0\.0001% 00:25:13）|- 带虚线下划线，hover 显示特殊浮窗：<br>- 间隔：交易对对应收取资金费率间隔小时<br>- 方向：<br>    - \>=0：买入 付款 卖出<br>    - \<0：卖出 付款 买入<br>- 资金费率： 下一次收取的资金费率<br>- 年化：\(24/间隔小时\)\* 下一次收取的资金费率 \* 365，显示成年化百分比<br>（静态文案）下一次资金的多头和空头头寸之间交换的支付利率。如果资金利率为正，多头支付空头。如果为负，空头支付多头。|
|24h成交量|24小时交易量，以USDT计价|大额数值用K/M/B缩写|
|未平仓合约|当前未平仓合约总价值，以USDT计价|大额数值用K/M/B缩写|
|24h最高价|24小时内最高成交价|无特殊交互|
|24h最低价|24小时内最低成交价|无特殊交互|



**2\.2\.4 数值格式化规则**

- 大额数值自动缩写：千（K）、百万（M）、十亿（B），保留两位小数。

- 示例：1,234,567 → 1\.23M；60,230,000 → 60\.23M。



**2\.2\.5 新币未开盘状态**

当用户选中尚未正式开启交易的合约（新币上线倒计时阶段）时，市场信息栏展示规则如下：



**（1）仍正常展示**

- 交易对名称（如 PEPEUSDC），字体加粗。

- 该交易对支持的最大杠杆倍数标签（如「50x」）。

- 点击交易对名称/杠杆区域，仍可弹出交易对切换浮窗（见 2\.3）。



**（2）其余字段统一显示占位符**

- 最新价、24h 涨跌幅、标记价格、指数价格、资金费率 / 倒计时、24h 成交量、未平仓合约、24h 最高价、24h 最低价，均显示 `--`。

- 占位符使用灰色字体，不展示涨跌颜色。

- 字段标签本身仍保留，不展示 tooltip 关联的实时数值。



**（3）开盘后的恢复**

- 合约到达开盘时间、行情数据可用后，市场信息栏各字段自动恢复为实时数据展示，无需用户手动刷新页面。

## 2\.3 交易对切换浮窗

**2\.3\.1 触发与关闭**

- 点击市场信息栏的交易对名称/杠杆区域，弹出浮窗。

- 浮窗定位在触发按钮正下方，左对齐。

- 再次点击触发按钮、或点击浮窗外部、或选择某个交易对后，浮窗关闭。

- 浮窗为模态行为：出现时页面其他区域不可交互，关闭后恢复。



**2\.3\.2 浮窗结构**

从上到下依次为：搜索框 → 分类标签栏 → 表头排序栏 → 行情列表。

---

**（1）搜索框**

- 位置：分类标签栏上方。

- 功能：支持模糊搜索交易对名称（如输入“BTC”可匹配“BTCUSDT”）。

- 交互：输入后实时过滤当前标签下的交易对列表；清空搜索词恢复完整列表。

- 异常：若搜索无匹配结果，列表区域显示空状态提示文案（如“暫無匹配結果”）。

---

**（2）分类标签栏**

- 支持横向滚动，适应标签数量扩展。

- 标签构成：

    - **固定标签（不可被后台移除）**：「收藏」「全部」「热门」，始终出现在标签栏最前面。

    - **动态标签**：由后台管理系统配置，前端根据返回的标签列表动态渲染。示例：新币上线、TradeFi、主流币、AI 等。

- 标签联动规则：

    - 后台下架某标签时，该标签直接隐藏。

    - 后台新增标签时，追加到标签栏末尾。

- 各标签行为：

    - **「全部」**：展示平台所有可交易交易对，按默认排序排列。搜索时范围为全部交易对。

    - **「收藏」**：展示用户已收藏的交易对。若无自选，显示空状态提示“暂無自選，快去添加吧”，并显示“编辑自选”按钮（铅笔图标），点击可进入批量删除、排序等操作。

    - **「热门」**：[“热门”标签的特殊排序与填充逻辑](https://ksgf73ukjokp.sg.larksuite.com/docx/SLB5dwR8eo5xy9xNLAFllX1Lg4b?from=from_copylink)

    - **动态标签**：仅展示带有该标签的交易对列表。

---

**（3）表头排序栏**

每列均支持点击排序，当前排序列的箭头高亮为黑色，其余为灰色。

|列名|排序能力|说明|
|---|---|---|
|合约|支持|按交易对名称字母排序|
|价格|支持|按最新价数值排序|
|24H涨跌幅|支持|按涨跌幅百分比排序|
|交易量|支持|按24h成交额排序|
|未平合约|支持|按未平仓合约价值排序|
|资金费率|支持|按当前资金费率排序|

- 每列排序支持升序/降序切换，默认排序为交易量降序。

- 排序仅在当前选中标签和搜索结果内生效。

---

**（4）行情列表**

列表项按行展示，每行一个交易对，包含以下字段：

|字段|说明|
|---|---|
|收藏星标|空心五角星图标，点击可收藏/取消收藏该交易对|
|合约名称|如 BTCUSDT，字体加粗|
|杠杆标签|如“100x”，显示该交易对最大杠杆倍数|
|最新价|当前最新成交价，保留适当小数位|
|24H涨跌幅|百分比显示，涨为绿色、跌为红色，带正负号|
|交易量|24h成交额，以USDT计价，大额使用K/M/B缩写|
|未平合约|未平仓合约总价值，以USDT计价，大额使用K/M/B缩写|
|资金费率|当前资金费率，带正负号|

- 点击某行交易对，切换当前页面的交易对为该合约，并关闭浮窗，页面数据（K线、订单簿、市场信息等）同步刷新。

---

**（5）数据更新策略**

- 浮窗内行情数据需实时或准实时更新，可采用 WebSocket 推送或定时轮询。

- 更新时仅刷新变化的字段，避免整列表重绘。

- 若连续多次更新失败，列表进入异常状态并提示用户。

---

## 2\.4 自选功能

- 在行情列表或交易对详情中，点击星形图标可添加/移除收藏。

- 收藏状态需持久化保存，登录后同步。

# 3\. 图表区域

## 3\.1 功能概述

图表区域为合约交易页面的核心可视化模块，支持四个子视图切换：K线图、深度图、合约详情、资金费率。默认展示K线图。



## 3\.2 子视图切换

- 顶部右侧提供四个切换按钮：图表、深度、详情、资金费率。

- 当前激活的视图按钮以深色背景高亮，其余为灰色。

- 点击按钮切换到对应视图，切换时保留当前交易对上下文。

- 四个视图共用同一区域，切换时仅显示当前视图，隐藏其余。

---

## 3\.3 K线图视图



**3\.3\.1 功能概述**

K线图区域集成第三方图表组件（如 TradingView），提供专业级K线图表分析功能。



**3\.3\.2 图表工具条**

**（1）时间周期切换**

- 位置：图表左上方。

- 可选周期：1分、5分、15分、30分、1小时、4小时、天、周、月。

- 当前选中周期以蓝色高亮显示。

- 点击周期按钮，图表切换至对应时间粒度。

    

**（2）K线自定义**

- 位于周期切换右侧，提供以下图标按钮：

    - 一个K线的图标：支持K线类型切换（如美国线、K线、空心K线、柱状图等）。

    - “技术指标”按钮：点击支持 TradingView 的热门指标



**（3）左侧纵向画图工具栏**

- 位于图表最左侧，提供垂直排列的快捷画图的工具栏：

    - 十字准线/光标切换。

    - 价格刻度切换。

    - 等\.\.\.



**（4）图表显示模式**

- 右侧提供百分比/对数坐标切换按钮：

    - “%”：切换为百分比坐标模式。

    - “log”：切换为对数坐标模式。

    - 当前激活模式高亮显示。

- 提供“自动”按钮，点击切换为自动坐标模式（默认激活）。

    

**（5）价格类型切换**

- “最新价格”下拉：可选择按最新价、标记价、指数价等切换图表价格显示基准。



**（6）配置记忆**

- 以上的 1、2、4、5 配置后，以记忆到缓存内的，用户切换交易对，仍显示对应的配置。



**3\.3\.3 图表主体**

**（1）OHLC 行情信息**

- 图表左上角叠加显示当前周期的开盘价、最高价、最低价、收盘价（或实时最新价）。

- 涨跌数值以绿色/红色区分。

    

**（2）K线主图**

- 以蜡烛图形式展示价格走势。

- 红色蜡烛表示下跌，绿色蜡烛表示上涨。

- 右侧纵轴显示价格刻度，动态适配当前可见价格范围。

- 横轴底部显示时间轴，自动根据周期格式化为对应日期/时间标签。

    

**（3）成交量副图**

- 主图下方展示成交量柱状图。

- 柱状图颜色与对应K线颜色一致（红跌绿涨）。

- 右侧纵轴显示成交量刻度。

- 成交量图顶部可叠加显示 SMA（移动平均线）数值。

    

**（4）最新价指示线**

- 图表上叠加一条水平虚线，标记当前最新成交价位置。

- 右侧纵轴上以标签形式显示最新价数值。

    

**（5）交互操作**

- 支持鼠标拖拽平移图表（水平/垂直方向）。

- 支持滚轮缩放图表（时间轴和价格轴同时缩放）。

- 支持十字光标：按住鼠标显示十字准线，并弹出浮层显示当前定位点的OHLC数据及时间。

- 图表上的绘图元素支持选中、拖拽修改、删除。

    

**（6）图表底部状态栏**

- 左侧显示时钟图标。

- 右侧显示当前时间（格式：17:35:52 \(UTC\+8\)），每秒实时更新。

- 时间右侧显示坐标模式快捷切换按钮：% \| log \| 自动。



**3\.3\.4 新币上线倒计时视图**

当当前交易对尚未开盘、尚无 K 线行情数据时，图表区域不展示 K 线图，改为整页展示距离开盘倒计时。

- 后端返回合约状态为「待开盘」或等价状态（如 `listingStatus = pending`），且当前时间早于 `openTime`。

- 开盘时间到达后，自动切换为正常 K 线图视图（见 3\.3\.1～3\.3\.3），无需用户操作。



**（2）顶部工具栏**

- 左侧：展示「新幣上線」标签（橙色）+ 当前交易对名称（如 PEPEUSDC·永續）。

- 右侧：保留「圖表 / 深度 / 詳情 / 資金費率」子视图切换按钮，切换逻辑与 3\.2 一致。



**（3）主内容区（居中展示）**

| 元素 | 说明 |
| --- | --- |
| 代币图标 | 展示该交易对的基础资产图标；无图标时可用首字母占位 |
| 标题 | 如「PEPE / USDC 永續合約」 |
| 副标题 | 如「新幣永續合約即將開啟交易，敬請期待」 |
| 倒计时标题 | 「距離交易開啟」 |
| 倒计时数字 | 天 / 时 / 分 / 秒 四格，等宽数字展示，每秒递减更新 |
| 开盘时间 | 展示官方开盘时间，含时区（如 2026-07-11 12:00:00 \(UTC\+8\)） |
| 合约信息 | 最高杠杆、结算币种、合约类型等静态配置信息 |



**（4）底部提示**

- 固定文案：「交易開啟前暫無行情數據 · 開盤後將自動切換至 K 線圖表」。



**（5）数据与交互**

- 倒计时由 `openTime` 与客户端当前时间差值计算，每秒刷新。

- 倒计时归零后，页面自动进入正常 K 线模式，并同步刷新市场信息栏、订单簿等区域。

## 3\.4 深度图视图



**3\.4\.1 功能概述**

以可视化方式展示当前交易对的买卖盘深度分布，帮助用户评估市场流动性。



**3\.4\.2 图表构成**



**（1）买盘区域（Bids）**

- 位于图表左侧，以绿色渐变填充区域表示买方挂单累计量。

- 横轴为累计挂单数量（以BTC等基础资产为单位），纵轴为价格。

- 曲线从中间向左侧延伸：越靠近当前价格的深度越厚，越远越薄。

    

**（2）卖盘区域（Asks）**

- 位于图表右侧，以红色渐变填充区域表示卖方挂单累计量。

- 曲线从中间向右侧延伸：越靠近当前价格的深度越厚，越远越薄。

    

**（3）中间价标识**

- 买卖盘曲线交汇处显示当前中间价格（最新成交价）。

- 中间价以标签形式居中展示，下方有一条垂直线指向价格位置。

    

**（4）坐标轴**

- 左侧纵轴显示数量刻度（如 0 / 100 BTC / 200 BTC）。

## 4\.3 订单簿视图

**4\.3\.1 表头**

- 三列布局：价格\(USDT\) \| 数量\(USDT\) \| 累计\(USDT\)

    - 数量 和 累计括号里面的 USDT，会随着交易单位类型切换，而变更成计价币种/交易币种

        

**4\.3\.2 卖单挂单区（Asks）**

- 位于订单簿上半部分，以红色系表示卖方挂单。

**4\.3\.4 买单挂单区（Bids）**

- 每一行背景以从右向左的绿色渐变条表示该价位累计深度的占比，规则同卖单：越靠近当前价累计量越大，渐变条越长；越远离越短。

- 数据更新方式同卖单。

    

**4\.3\.5 底部工具栏**

订单簿底部固定一行操作项：



**（1）精度选择**

    - 第二档：基础精度 × 10（如“0\.1”）。

    - 第四档：基础精度 × 1000（如“10”）。

- 每个档位的显示文本即为计算后的精度数值（如“0\.01”“0\.1”“1”“10”）。

- 用户选择任意档位后，订单簿内的挂单价格将按该精度进行聚合显示，同时订单簿重新渲染。

- 下拉菜单采用绝对定位，点击外部自动关闭。

- 若交易对价格精度发生变更（例如切换交易对），精度档位自动重新计算，并重置为第一档（基础精度）。



**（2）计价资产选择**

- 显示当前计价单位（如“USDT”），旁边有下拉三角图标。

- 切换后订单簿的价格显示单位、数量单位同步变更；同时影响最新成交记录中的数量单位。

- 下拉菜单同样采用绝对定位，点击外部自动关闭。

# 5\. 交易终端

## 5\.1 功能概述

交易终端位于页面右侧，是用户执行下单操作的核心区域。支持开仓/平仓切换、限价/市价下单、止盈止损设置、保证金模式与杠杆调整，并提供下单确认与结果反馈。



## 5\.2 仓位模式与杠杆



**5\.2\.1 仓位模式切换**

**（1）独立设置原则**

- 每个交易对的仓位模式（全仓/逐仓）独立设置，互不影响。

- 切换交易对时，仓位模式自动切换为该交易对已保存的设置；若首次交易该合约，默认全仓。

    

**（2）切换触发**

- 点击仓位模式按钮（显示当前模式，如“全仓”），弹出保证金模式选择弹窗。

- 弹窗内两个卡片式选项：全仓 / 逐仓。

- 选中卡片高亮

- 弹窗内说明文案：

    - 全仓：保证金资产相同的全仓仓位共享该资产的全仓保证金。在强平事件中，交易者可能损失全部该保证金和该保证金资产下的所有全仓仓位。

    - 逐仓：一定数量保证金被分配到仓位上。如果仓位保证金亏损到低于维持保证金的水平，仓位将被强平。

    - 底部提示：“调整保证金模式仅对当前合约生效”。

- 点击“确认”保存设置并关闭弹窗。

    

**（3）切换限制**

- **当前交易对存在未平仓持仓时，不允许切换仓位模式。**

- 若用户在持有仓位时尝试切换，点击“确认”后弹窗不关闭，并弹出错误提示：“当前合约存在持仓，无法切换保证金模式。”

- 无持仓时正常切换。



**5\.2\.2 杠杆倍数设置**

**（1）多空杠杆独立设置**

- 每个交易对的**做多杠杆**和**做空杠杆**可以**独立设置**，互不影响。

- 用户可为同一交易对的不同方向设置不同的杠杆倍数（如做多 100x，做空 50x）。

- 切换交易对时，多空杠杆分别恢复为该交易对已保存的设置；若首次交易该合约，默认均为后台配置的最低杠杆倍数（或用户上次在该交易对设置的值）。

    

**（2）杠杆阶梯由后台配置决定**

- 每个交易对的杠杆配置独立，由后台风控系统配置。

- 杠杆不是随意输入的任意值，而是由后台配置的**杠杆阶梯**决定。

- 后台配置包括：

    - 该交易对允许的最大杠杆倍数（如 100x）。

    - 该交易对多空方向是否支持不同杠杆阶梯（通常一致，但支持独立配置）。

    - 每个杠杆阶梯对应的最大可开仓位名义价值（即仓位价值上限）。

- 示例阶梯（多空可共用同一套阶梯，也可独立配置）：

|杠杆倍数|最大名义价值（USDT）|
|---|---|
|100x|≤ 50,000|
|75x|≤ 100,000|
|50x|≤ 200,000|
|25x|≤ 500,000|
|10x|≤ 1,000,000|
|5x|≤ 2,000,000|
|1x|无上限|

- 前端从后台接口获取当前交易对的杠杆阶梯配置，渲染滑块可用档位。

    

**（3）杠杆调整弹窗**

- 点击交易终端顶部的杠杆倍数按钮，弹出杠杆调整弹窗。

- 弹窗内分为两个区域：

    

**【做多杠杆】区域：**

- 显示当前做多杠杆倍数。

- 步进输入器（\- / 输入框 / \+）和滑块，档位由后台杠杆阶梯配置决定。

- 下方显示当前做多杠杆阶梯下的最大可开名义价值。

    

**【做空杠杆】区域：**

- 显示当前做空杠杆倍数。

- 步进输入器和滑块，与做多区域交互一致。

- 下方显示当前做空杠杆阶梯下的最大可开名义价值。

    

- 两个区域的杠杆可分别调整，互不影响。

- 底部“确认”按钮，点击保存多空杠杆设置并关闭弹窗。

    

**（4）可开仓位的联动计算**

- 用户选择不同杠杆倍数时，底部数据面板的“可开”数值联动更新。

- 计算逻辑按**方向**分别计算：

    - **做多可开**：做多杠杆阶梯对应的仓位价值上限 \- 当前该合约**做多方向已开名义价值之和**（含挂单冻结保证金对应的名义价值）。若结果 ≤ 0，则显示 0。

    - **做空可开**：做空杠杆阶梯对应的仓位价值上限 \- 当前该合约**做空方向已开名义价值之和**（含挂单冻结保证金对应的名义价值）。若结果 ≤ 0，则显示 0。

- 底部数据面板的“可开”字段按方向分别展示：

    - 开多模式下：显示做多可开。

    - 开空模式下：显示做空可开。

        

**（5）有持仓时的杠杆调整限制**

- **做多杠杆**：当前合约存在**做多仓位**时，做多杠杆只允许调高，不允许调低。无做多仓位时可自由调整。

- **做空杠杆**：当前合约存在**做空仓位**时，做空杠杆只允许调高，不允许调低。无做空仓位时可自由调整。

- 杠杆调高后，对应方向仓位的杠杆标记同步更新为新高倍数。

- 调低时，滑块或输入框禁止操作低于当前杠杆倍数的值，并提示：

\> “当前合约存在持仓，无法调低杠杆。请先平仓后再操作。”



**（6）杠杆调整的生效范围**

- 杠杆调整仅更新对应方向持仓和挂单的**杠杆标记信息**，用于展示和后续开仓限额计算。

- **实际杠杆率** = 持仓名义价值 / 保证金，由保证金和持仓金额的实际比例决定，不受杠杆标记直接影响。

- 调整后，前端以下位置的杠杆倍数显示同步刷新：

    - 交易终端顶部的杠杆倍数按钮。

    - “当前持仓”表格中对应方向仓位的杠杆标签。

    - “当前委托”表格中对应方向挂单的杠杆标签。

## 5\.3 开仓/平仓切换

**（1）切换按钮**

- 两个互斥按钮：「开仓」（默认选中，白底加阴影）、「平仓」（灰色文字）。

- 切换至平仓时，止盈止损勾选框区域（跟随止盈止损）完全隐藏。

- 切换回开仓时，止盈止损勾选框区域恢复显示。

    

**（2）下单按钮联动**

- 开仓模式：底部显示绿色「开多」和红色「开空」两个按钮。

- 平仓模式：底部按钮根据当前持仓方向动态变化：

    - 持多仓时：显示橙色「市价平多」和「限价平多」。

    - 持空仓时：显示橙色「市价平空」和「限价平空」。

---

## 5\.4 订单类型与下单参数配置



三种订单类型（限价、市价、止盈止损）的下单参数各有不同，以下按订单类型逐一说明其参数构成、输入框交互与前端验证规则。



### 5\.4\.1 限价单

**（1）订单参数**

|参数|说明|
|---|---|
|下单价格|用户手动输入期望的委托价格|
|下单数量|用户输入开仓/平仓数量|
|订单有效期|GTC / FOK / IOC（详见 5\.7）|
|止盈/止损（可选）|勾选后可附加跟随止盈止损条件单（详见 5\.8）|



**（2）价格输入框交互**

- 输入框默认显示当前最新成交价，可编辑。

- 输入框右侧显示计价单位（如 USDT）。

- 输入框右侧有「最优价」按钮（功能详见 5\.5\.1）。



**（3）数量输入框交互**

- 手动输入数量。

- 右侧显示下单单位（默认 USDT），点击可切换为 BTC 等基础资产。

- 切换单位后，数量值按当前市价自动换算并清空原有输入。

- 数量滑块：0% / 25% / 50% / 75% / 100%，拖动或点击档位自动填入对应比例的可用余额。

- 可用余额展示：格式“12,450\.00 USDT”，旁有“\+”按钮（充币弹窗入口）。

    

**（4）前端验证规则**

**价格验证：**

- 只能输入数字，禁止字母和特殊字符。

- 价格必须 \> 0。

- 小数精度：不超过当前交易对的价格精度（如 BTCUSDT 精度 0\.1，则最多输入一位小数）。

    

**数量验证：**

- 只能输入数字。

- 小数精度：

    - 单位为 USDT（计价资产）时：精度与交易对价格精度一致。

    - 单位为基础资产（如 BTC）时：精度遵循后台配置的下单最小单位。

- 数量 × 市价（名义价值）不可超过当前杠杆阶梯的最大可开仓位金额（含已持仓和挂单占用的额度）。

- 数量不可小于该交易对的最小交易单位。

---

### 5\.4\.2 市价单



**（1）订单参数**

|参数|说明|
|---|---|
|下单价格|默认为“市价”，用户无需输入|
|下单数量|用户输入开仓/平仓数量|
|订单有效期|市价单默认 IOC（立即成交或取消），不可更改|
|止盈/止损（可选）|勾选后可附加跟随止盈止损条件单（详见 5\.8）|



**（2）价格展示**

- 价格输入框置灰不可编辑，居中显示“市价”灰色文字。

- 价格单位（如 USDT）隐藏。

- 「最优价」按钮隐藏。

    

**（3）数量输入框交互**

- 与限价单的数量输入框完全一致。

- 数量滑块、可用余额展示、单位切换均保持一致。

    

**（4）前端验证规则**

**数量验证：**

- 与限价单的数量验证规则一致（精度、最大可开、最小交易单位）。

---

### 5\.4\.3 止盈止损单

**（1）功能定位**

- 止盈止损是一种**条件订单**：用户预设触发价格，当市场价格达到时，系统自动提交开仓或平仓委托。

- 支持**开仓（开多/开空）和平仓（平多/平空）**两种方向，方向由用户通过下单按钮主动选择。

- 与限价/市价单上的“止盈/止损勾选框”（跟随止盈止损，见 5\.8）是**两个不同功能**。

    

**（2）方向选择**

止盈止损单的**开多/开空/平多/平空**方向由用户通过底部下单按钮主动选择，系统不做自动判断。



- 开仓时：用户先选择「双向」或「单向」模式，设置触发价格后，点击底部绿色「开多」或红色「开空」按钮提交订单。

- 平仓时：根据当前持仓方向，底部显示「平多」或「平空」按钮，用户点击提交。

    

**（3）双向/单向切换**

止盈止损单支持两种模式切换：「双向」与「单向」，由用户在下单前自行选择。

|模式|含义|适用场景|
|---|---|---|
|双向|同时设置两个触发价格，分别对应两个不同的触发条件|用户想同时预设两个入场/出场条件|
|单向|仅设置一个触发价格|用户只想在一个特定价位入场/出场|



**切换交互：**

- 默认选中「双向」。

- 两个按钮互斥切换，选中项高亮（黑色文字\+下划线），未选中为灰色。

- 切换时，下方输入区动态变化。

---

**（4）开仓止盈止损**



用户判断市场突破某个关键价位后将继续上涨或下跌，希望等价格到达目标位时自动开仓入场。**用户自行选择做多或做空方向。**



**【双向模式】**

输入区显示两个触发价格设置区域：

|设置项|说明|
|---|---|
|触发价格（上方）|输入框上方文案为「触发价格」|
|触发价格（下方）|输入框上方文案为「触发价格」|



- 用户可为两个触发价格分别设置不同的数值。

- 用户可只填一个触发价（另一个留空），即仅设置一个方向的条件单。

- 每个触发价格输入框右侧内嵌下拉选择触发价格类型：最新成交价（默认）/ 标记价格 / 指数价格。

- 每个触发价格下方可选择委托方式：市价（默认）/ 限价。

    - 若选限价，展开委托价格输入框。

        

**方向由用户选择：**

- 用户填写触发价格后，点击底部「开多」或「开空」按钮提交订单。

- 若点击「开多」：两个触发条件均用于开多方向。

- 若点击「开空」：两个触发条件均用于开空方向。

    

**【单向模式】**



输入区仅显示一个触发价格设置区域：

|设置项|说明|
|---|---|
|触发价格|输入框上方文案为「触发价格」|



- 用户设置一个触发价格。

- 触发价格输入框右侧内嵌下拉选择触发价格类型：最新成交价（默认）/ 标记价格 / 指数价格。

- 下方可选择委托方式：市价（默认）/ 限价。

    

**方向由用户选择：**

- 用户填写触发价格后，点击底部「开多」或「开空」按钮提交订单。

- 点击「开多」→ 触发后自动开多。

- 点击「开空」→ 触发后自动开空。

---

**（5）平仓止盈止损**



用户已持有仓位，希望预设止盈/止损出场条件。**平仓方向由当前持仓方向决定，止盈/止损由触发价与持仓成本的关系自动判定。**



**【双向模式】**

输入区显示两个设置区域：

|设置项|说明|
|---|---|
|止盈（上方）|输入框上方文案为「止盈」；设置止盈触发价格|
|止损（下方）|输入框上方文案为「止损」；设置止损触发价格|



- 用户分别设置止盈触发价和止损触发价。

- 每个触发价格输入框右侧内嵌下拉选择触发价格类型。

- 每个触发价格下方可选择委托方式：市价（默认）/ 限价。

    

**方向自动确定：**

- 当前持仓为多仓时，底部按钮为「平多」。

- 当前持仓为空仓时，底部按钮为「平空」。

- 止盈/止损的判定由触发价与持仓成本的关系自动决定，用户无需手动选择。

    

**【单向模式】**

输入区仅显示一个设置区域：

|设置项|说明|
|---|---|
|触发价格|输入框上方文案为「触发价格」|

- 用户仅设置一个触发价格。

- 触发价格输入框右侧内嵌下拉选择触发价格类型。

- 下方可选择委托方式：市价（默认）/ 限价。

    

**方向自动确定，止盈/止损自动判定：**

- 平仓方向由当前持仓方向决定（持多→平多，持空→平空），底部按钮自动显示。

- 系统根据触发价与持仓开仓均价的关系自动判定止盈还是止损：

    - 持多仓时：触发价 \> 持仓开仓均价 → 止盈；触发价 \< 持仓开仓均价 → 止损。

    - 持空仓时：触发价 \< 持仓开仓均价 → 止盈；触发价 \> 持仓开仓均价 → 止损。

- 用户点击底部「平多」或「平空」按钮提交订单。

---

**（6）数量输入**



- 双向和单向模式共用同一个数量输入框。

- 交互逻辑与限价单的数量输入框完全一致（详见 5\.4\.1）。

- 在双向模式下，若两个触发价格均填写且生效，数量按同一数量分别生效（即两个条件单各自使用相同的数量）。

---

**（7）前端验证规则**

**触发价验证：**

|场景|验证规则|
|---|---|
|开仓双向 \+ 开多|两个触发价均须 \> 当前最新成交价|
|开仓双向 \+ 开空|两个触发价均须 \< 当前最新成交价|
|开仓单向 \+ 开多|触发价须 \> 当前最新成交价|
|开仓单向 \+ 开空|触发价须 \< 当前最新成交价|
|平仓双向（持多）|止盈触发价 \> 持仓开仓均价；止损触发价 \< 持仓开仓均价|
|平仓双向（持空）|止盈触发价 \< 持仓开仓均价；止损触发价 \> 持仓开仓均价|
|平仓单向（持多）|触发价须 ≠ 持仓开仓均价|
|平仓单向（持空）|触发价须 ≠ 持仓开仓均价|

- 所有触发价不能等于当前最新成交价。

- 触发价小数精度须符合交易对价格精度。

    

**委托价验证（仅限价委托时）：**

|方向|验证规则|
|---|---|
|开多|委托价 ≥ 触发价|
|开空|委托价 ≤ 触发价|
|平多|委托价 ≥ 触发价|
|平空|委托价 ≤ 触发价|

- 委托价小数精度同触发价。



**数量验证：**

- 与限价单的数量验证规则一致（精度、最大可开、最小交易单位）。

- 开仓止盈止损需额外检查保证金是否充足。

---



**（8）止盈止损订单的生命周期**

- 已提交但未触发：显示在“当前委托”列表中，支持手动撤单和修改。

- 触发后：系统自动提交委托，止盈止损订单状态变为“已完成”，后续成交情况在“历史委托”中查看。

    - 如果是双向委托单，另外一边成交了，系统取消订单，状态为“已取消”。

- 修改时：点击编辑图标回到设置状态，调整参数后重新提交。

## 5\.5 最优价功能（仅限价单可用）



**（1）触发方式**

- 限价单价格输入框右侧的「最优价」按钮。

    

**（2）按钮样式**

- 按钮底部有虚线下划线，表示可交互。

- 鼠标悬停时弹出 tooltip 解释：

\> “最优价 \(BBO\) 可为限价单快速设置订单簿上的最佳买入价或最佳卖出价。同向价指当前交易方向的最佳价格，对手价指相反交易方向的最佳价格。1 指订单表最佳价格，5 指订单表排序第 5 的最佳价格。”



**（3）交互逻辑**

- **点击「最优价」按钮**：价格输入框切换为下拉选择框，显示以下选项：

    - 对手价 1（默认选中）

    - 对手价 5

    - 同向价 1

    - 同向价 5

- 选择后，下拉框保持显示所选文字（不自动切换回输入框），价格单位（USDT）隐藏。

- **再次点击「最优价」按钮**：下拉框切换回普通输入框，并将当前选中选项对应的价格填入输入框，价格单位恢复显示。

    

**（4）价格映射**

- 对手价 1：订单簿对手方（买/卖）第 1 档价格。

- 对手价 5：订单簿对手方第 5 档价格。

- 同向价 1：订单簿同方向第 1 档价格。

- 同向价 5：订单簿同方向第 5 档价格。

- 映射关系由订单簿实时数据动态计算。

## 5\.6 订单有效期



**（1）可选类型**

- **GTC**（一直有效直至取消）：订单将持续有效直到完全成交或被用户手动取消。

- **FOK**（全部成交否则取消）：订单必须立即全部成交，否则立即取消。

- **IOC**（立即成交或取消）：订单必须立即成交，可部分成交，未成交部分立即取消。

    

**（2）适用订单类型**

- 限价单：支持 GTC / FOK / IOC。

- 市价单：默认 IOC，不可更改。

- 止盈止损单：默认 GTC（一直有效直至触发），不可更改。

    

**（3）后端同步**

- 提交订单时，将有效期类型（GTC / FOK / IOC）作为订单参数传给后端。

- 后端处理逻辑：

    - GTC：持续挂单直到成交或取消。

    - FOK：

    - IOC：

## 5\.7 跟随止盈止损（勾选框）



**（1）功能定位**

- 该功能为**附加在限价单/市价单主订单上的条件单**，非独立订单类型。

- 仅在**开仓模式**下可用，平仓模式下隐藏。

- 主订单成交后，止盈/止损条件单自动激活。

    

**（2）触发方式**

- 勾选“止盈/止损”复选框，展开止盈/止损输入区。

- 取消勾选则收起。

    

**（3）复选框样式与说明**

- 文字标签“止盈/止损”，标签下方有虚线下划线。

- 鼠标悬停时显示 tooltip：

\> “止盈止损是指下单时提前为该订单预设止盈价及止损价，以锁定利润或控制损失。当订单完全成交时，系统将自动进行止盈止损委托。”



**（4）输入区结构**

- 止盈区域（上排）：止盈触发价格输入框 \+ 触发价格类型下拉。

- 止损区域（下排）：止损触发价格输入框 \+ 触发价格类型下拉。

- 预估收益/亏损实时显示。

    

**（5）触发逻辑**

- 主订单完全成交后，系统自动根据设置的止盈/止损触发价挂出止盈/止损条件单。

- 若主订单部分成交，止盈止损条件单按成交比例激活。

- 若主订单被取消，止盈止损条件单不激活。

    

**（6）验证规则**

- 止盈触发价：做多时须高于开仓价；做空时须低于开仓价。

- 止损触发价：做多时须低于开仓价；做空时须高于开仓价。

- 其余验证规则与止盈止损订单类型一致（见 5\.4\.3）。

## 5\.8 开仓/平仓下单按钮与确认



**5\.8\.1 功能概述**

下单确认功能用于在用户提交订单前进行二次确认，防止误操作。支持对限价单、市价单、止盈止损单分别弹出确认弹窗，展示该笔订单的核心交易信息及预估爆仓价。用户确认后才真正提交订单；若关闭弹窗则取消本次下单。



同时提供“不再提示”选项，勾选后该类型订单将不再显示二次确认，并持久化保存该偏好。用户随时可在右侧偏好设置抽屉中重新开启或关闭各类订单的确认提示。



**5\.8\.2 下单按钮**



**（1）开仓模式**

- 两个按钮并排：

    - 绿色按钮「开多」：以做多方向开仓。

    - 红色按钮「开空」：以做空方向开仓。

- 点击任一按钮，拉起对应订单类型的下单确认弹窗（若用户已在偏好设置中关闭该类型二次确认，则直接提交订单）。

    

**（2）平仓模式**

- 根据当前持仓方向动态显示按钮：

    - 持多仓时：显示橙色「市价平多」和「限价平多」。

    - 持空仓时：显示橙色「市价平空」和「限价平空」。

- 用户输入平仓价格（限价）或直接点击市价按钮后，拉起确认弹窗。

    

**（3）止盈止损单的下单按钮**

- 止盈止损单作为独立订单类型，在开仓模式下显示绿色「开多」和红色「开空」；在平仓模式下显示橙色平仓按钮。

- 点击后拉起止盈止损单专用的确认弹窗。

---

**5\.8\.3 下单确认流程**



1. 用户在下单区点击下单按钮（开多/开空/平多/平空）后，不直接提交订单，而是拉起对应的下单确认弹窗。

2. 弹窗内展示该笔订单的关键交易字段，用户核对无误后点击确认按钮，订单才正式提交至交易引擎。

3. 点击弹窗右上角关闭按钮（×），或止盈止损弹窗中的「取消」按钮，则关闭弹窗，不提交订单。

4. 弹窗底部提供一个复选框：“不再为\{订单类型\}提示二次确认”。

    - 勾选后点击确认，本次正常下单，同时系统记录该偏好；后续同类型订单将跳过确认弹窗直接下单。

# 8\. 底部状态栏

## 8\.1 功能概述

底部状态栏固定悬浮在页面视口最下方，提供系统连接状态、网络延迟、重要公告轮播及社交媒体入口。该信息条始终可见，不随页面内容滚动而隐藏。



## 8\.2 左侧：系统连接状态

**（1）信号强度图标**

信号强度以四格信号图标展示，根据当前网络延迟（WebSocket 或 API 请求的平均往返时延）动态计算格数。格数计算方式如下：

|延迟范围|信号格数|图标颜色|状态说明|
|---|---|---|---|
|≤ 500ms|四格全亮|绿色|连接质量优秀|
|500ms \< 延迟 ≤ 1000ms|三格亮起|绿色|连接质量良好|
|1000ms \< 延迟 ≤ 2000ms|两格亮起|橙色|连接质量一般，可能存在轻微延迟|
|\> 2000ms|仅一格亮起|红色|连接质量差，延迟较高|
|连接完全断开（无响应/超时）|零格（四格全部空白）|灰色|无连接|

- 亮起的格子为实心填充对应颜色，未亮起的格子为浅灰色空心轮廓。

- 延迟数据由前端定时测量（如每 5 秒发送一次心跳包并计算往返时间），取最近三次测量的平均值作为当前延迟值。

    

**（2）连接状态文字**

- 连接正常时：显示“已连接”，绿色加粗字体。

- 连接断开时：显示“连接断开”，红色字体，并伴随闪烁动画提示用户注意。

- 断连超过一定时间阈值（如 30 秒）仍未恢复，文字保持红色常亮不再闪烁。

    

**（3）网络延迟数值**

- 显示当前网络延迟毫秒数（如“154ms”），灰色字体。

- 数值随每次延迟测量结果更新。

- 连接断开时显示“\-\-ms”，灰色字体。

    

**（4）图标、文字与延迟的整体联动**

|连接状态|信号图标|状态文字|延迟显示|
|---|---|---|---|
|正常（≤500ms）|四格绿色|已连接（绿色）|如“154ms”（灰色）|
|正常（500\-1000ms）|三格绿色|已连接（绿色）|如“780ms”（灰色）|
|正常（1000\-2000ms）|两格橙色|已连接（绿色）|如“1520ms”（灰色）|
|正常（\>2000ms）|一格红色|已连接（绿色）|如“2350ms”（灰色）|
|断开|零格灰色空白|连接断开（红色闪烁）|\-\-ms（灰色）|

## 8\.3 中间：公告轮播

- 展示最新的 6 条平台公告文案（由后端配置并下发）。

- 轮播方式：公告文字在区域内从右至左无限循环滚动，鼠标悬停时滚动暂停，移开后继续。

- 每条公告均为可点击文字，点击后在新标签页或弹出公告详情页展示完整内容。

- 若无公告或加载失败，该区域显示暂无公告的默认文案，不滚动。

    

## 8\.4 右侧：社交媒体图标

- 展示三个社交媒体快捷入口图标：X（Twitter）、Discord、Telegram。

- 鼠标悬停时图标颜色加深，点击后在新标签页打开对应的官方社交链接。

- 图标顺序固定为 X → Discord → Telegram，不可变更。

- 链接地址由后台配置或前端统一维护。

    

## 8\.5 数据更新与异常处理
