# 排行榜

> # **【重要说明】**
>
> 本文档为 Web 端排行榜页面及后台「排行榜影子配置」的交互与功能需求说明，供产品、UI 及前后端开发参考。
>
> 排行榜核心统计口径、影子账户调度算法、风控排除规则等底层逻辑以产研团队及合作方系统规则为准。本文档描述页面信息展示、筛选排序、后台配置与交互行为；如有差异，以实际系统规则为准。

---

# 一、Web 端排行榜页

原型路径：`perp_dex/排行榜.html`

## 1\.1 功能定位

展示平台交易者的综合表现排名，支持按多个维度查看与排序。用户可了解自身在全站榜单中的位置，并按钱包地址搜索其他交易者。

当前支持四个排名维度：

|维度|字段|说明|
|---|---|---|
|收益金额|PnL|统计周期内已实现盈亏金额（USDT）|
|收益率|ROI|统计周期内投资回报率（%）|
|交易额|Volume|统计周期内合约成交额（USDT 名义价值）|
|积分|Points|统计周期内获得的平台积分|

## 1\.2 页面结构

页面自上而下依次为：

|区域|说明|
|---|---|
|顶部导航栏|全站统一导航，当前页「排行榜」高亮|
|页面标题区|标题、统计周期切换（7D / 30D）|
|我的数据摘要|当前用户排名、收益、交易额|
|地址搜索|按钱包地址模糊搜索|
|排行榜表格|排名列表，支持列头排序|
|底部说明与分页|展示规则说明、分页控件|

## 1\.3 顶部导航栏

导航栏整体功能（菜单、语言切换、链信息、用户钱包等）详见《统一顶部导航栏》文档；本页仅补充「排行榜」菜单项为当前选中态。

## 1\.4 统计周期切换

- 位于页面标题右侧，分段切换：**7D**（默认选中）、**30D**。

- 切换后重新拉取对应周期内的排行榜数据，并重置当前排序维度与分页至第一页。

- 我的排名摘要区同步刷新。

## 1\.5 我的数据摘要

登录用户展示紧凑信息卡，未登录时可隐藏或展示「登录后查看我的排名」引导。

|字段|说明|
|---|---|
|我的排名|当前用户在**当前排序维度**下的名次，如 #1,240|
|我的收益|当前周期 PnL，涨绿跌红，带正负号|
|我的交易额|当前周期合约成交额（USDT），琥珀色强调|

- 「我的排名」随用户点击的排序列变化：例如按交易额排序时，展示的是交易额维度下的名次。

## 1\.6 地址搜索

- 占位文案：「按錢包地址搜索...」

- 支持钱包地址模糊匹配，实时或点击搜索后过滤列表。

- 清空搜索恢复完整榜单。

## 1\.7 排行榜表格

### 1\.7\.1 列定义

|列名|对齐|说明|
|---|---|---|
|排名|居中|当前排序维度下的名次；Top 1–3 使用徽章样式（黑/深灰/浅灰）|
|交易者|左对齐|钱包地址缩写或昵称|
|收益金额 (PnL)|右对齐|带正负号，盈利绿色，亏损红色|
|收益率 (ROI)|右对齐|百分比，涨绿跌红|
|交易额 (Volume)|右对齐|USDT 名义成交额，大额可用 K/M/B 缩写，琥珀色强调|
|积分|右对齐|整数展示|

### 1\.7\.2 排序规则

- 表头 PnL、ROI、交易额、积分均可点击排序。

- 默认排序维度：**收益率 (ROI)** 降序（当前选中列高亮为黑色）。

- 点击列头切换升序/降序；再次点击另一列则切换排序维度。

- 排名列数值随当前排序维度重新计算。

### 1\.7\.3 行交互

- 行 hover 浅灰背景。

- Top 3 排名徽章与后续纯数字排名样式区分。

### 1\.7\.4 数据展示规则

- PnL：保留两位小数，正数带「+」前缀。

- ROI：保留两位小数，带「%」。

- 交易额：按平台统一金额格式；建议使用 K/M 缩写（如 $128.6M）。

- 积分：千分位分隔，无小数。

## 1\.8 底部说明与分页

**（1）说明文案**

- 「排名僅展示前 1,000 名交易者」

- 「數據每 5 分鐘更新一次」

**（2）分页**

- 标准分页控件：上一页、页码、下一页。

- 每页条数由后台配置（原型默认展示多行 + 分页示意）。

## 1\.9 特殊状态

|状态|表现|
|---|---|
|未登录|可浏览榜单；我的数据摘要引导登录|
|加载中|表格骨架屏或 Loading|
|无搜索结果|空状态「未找到匹配地址」|
|接口失败|错误提示 + 重试|

# 二、后台管理：排行榜影子配置

原型路径：`perp_dex/admin/排行榜后台.html`

## 2\.1 功能定位

运营后台用于配置**影子账户**（Shadow Account），使其在指定榜单的目标排名区间内竞争，以维持榜单活跃度与竞争氛围。影子账户由调度引擎按配置自动调整交易行为，直至进入目标区间或生命周期结束。

本模块支持两类**目标榜单**：

|目标榜单|说明|
|---|---|
|全平台|影子账户在全站用户排名中竞争目标名次|
|平台活动|影子账户仅在**已报名该交易排名活动**的用户子榜单中竞争目标名次|

每个影子账户可独立配置 PnL、ROI、交易额、积分四个维度的目标排名区间，以及生命周期与人设参数。

## 2\.2 页面结构

|区域|说明|
|---|---|
|侧边栏|运营模块导航，当前项「排行榜影子配置」|
|页头|标题 + 功能说明|
|筛选条|多条件过滤影子账户池|
|影子账户池列表|表格展示 + 新增入口|
|新增/编辑弹窗|全量参数配置|

## 2\.3 筛选条

支持与列表联动筛选，点击「筛选」应用条件，「重置」清空。

|筛选项|说明|
|---|---|
|地址模糊搜索|影子钱包地址关键词|
|目标榜单|全部 / 全平台 / 平台活动|
|活动名称|选择「平台活动」后启用；按活动名模糊查询|
|PnL 目标排名|目标区间 Min–Max（名次）|
|ROI 目标排名|同上|
|交易额目标排名|同上|
|积分目标排名|同上|
|账户状态|全部 / 活跃 / 衰退 / 下线|

- 目标榜单决定筛选范围：「平台活动」须结合活动名称进一步缩小结果。

## 2\.4 影子账户池列表

### 2\.4\.1 表头字段

|列名|说明|
|---|---|
|影子地址|钱包地址缩写|
|目标榜单|全平台徽章，或「平台活动 + 活动名称」|
|目标排名要求|四行展示：PnL / ROI / Volume / Points 各自目标区间|
|当前状态|活跃 / 衰退 / 下线|
|生命周期|进度条 + 文案（如 Day 10 / 30）|
|当前数据|PnL、ROI、交易额、积分当前值|
|操作|编辑、下线|

### 2\.4\.2 状态说明

|状态|含义|
|---|---|
|活跃 (Active)|正在向目标区间调度|
|衰退 (Decay)|已触发下线，数据逐步回落|
|下线 (Offline)|已停止调度|

### 2\.4\.3 操作

- **新增影子账户**：打开配置弹窗。

- **编辑**：回填该账户配置。

- **下线**：二次确认后进入衰退流程，周期结束时由继承逻辑处理。

## 2\.5 新增/编辑影子账户弹窗

### 2\.5\.0 目标榜单

双选项切换：

|选项|行为|
|---|---|
|全平台（默认）|影子账户在全站榜单竞争|
|平台活动|展示「选择交易排名活动」下拉，必选一项|

活动下拉数据来源：活动上架管理系统中已发布、含交易排行榜模块的活动（如 ForX嘉年华交易大赛、BTC 永续交易量冲榜等）。

底部说明文案随选项变化：

- 全平台：说明将在全站 PnL / ROI / 交易额 / 积分排名中竞争。

- 平台活动：说明仅针对**已报名该活动**的用户子榜单竞争。

保存校验：选择「平台活动」但未选活动时，提示「请选择交易排名活动」。

### 2\.5\.1 维度排名区间目标

四个维度各自独立配置目标名次区间，下拉档位一致：

|档位|名额|
|---|---|
|Top 1 – 3|3|
|Top 4 – 10|7|
|Top 11 – 20|10|
|Top 21 – 50|30|
|Top 51 – 100|50|
|Top 101 – 200|100|
|Top 201 – 500|300|

|维度|说明|
|---|---|
|收益金额 (PnL Amount)|盈亏金额排名目标|
|收益率 (ROI %)|回报率排名目标|
|交易额 (Volume)|合约成交额（USDT 名义价值）排名目标|
|获得积分 (Points)|积分获取排名目标|

- 系统将校验同一目标榜单 + 同一维度下区间余量，避免影子账户目标重叠。

- 交易额维度说明：按合约成交额（USDT 名义价值）统计，与 Web 端排行榜 Volume 列口径一致。

### 2\.5\.2 生命周期与人设

|字段|说明|
|---|---|
|活跃时长 (天)|影子账户本周期运行天数，如 30|
|自动继承者逻辑|勾选后，周期结束自动生成新地址承接相同参数|

### 2\.5\.3 保存

- 「保存并应用」：校验通过后提交调度引擎同步。

- 「取消」：关闭弹窗不保存。

## 2\.6 弹层叠放规范（后台）

后台弹窗层级简单，主要为居中配置弹窗；若后续扩展嵌套选择器，须保证后打开层覆盖先打开层，避免选项被遮挡。

# 三、Web 端与后台的数据关系

|项目|Web 排行榜页|后台影子配置|
|---|---|---|
|统计维度|PnL、ROI、交易额、积分|同左，均可设目标区间|
|榜单范围|全平台公开榜单|全平台 或 指定活动报名用户子榜|
|交易额口径|周期内合约成交额（USDT）|同左|
|排名更新频率|每 5 分钟（前端展示）|调度引擎按目标区间实时/准实时调整|
|Top 展示上限|前 1,000 名|目标区间最大至 Top 500|

# 四、附录：活动榜单与全平台榜单的区别

**全平台榜单**

- 统计所有符合条件的真实用户 + 影子账户。

- Web 端默认展示此榜单。

**平台活动榜单**

- 仅统计已报名该活动的用户（+ 绑定该活动目标的影子账户）。

- 活动详情页内的交易大赛排行榜与后台「平台活动」目标榜单对应。

- 同一影子账户不可同时以不同活动目标冲突配置（同一地址仅一套生效配置）。
