# APP 端奖励中心

# 0. 页面概述

## 0.1 功能定位

奖励中心是 APP 底部 Tab「奖励」对应的主页面，承担以下职责：

- **活动引流**：展示进行中活动总奖池与一条热门活动推荐，引导用户进入活动中心参与。
- **权益总览**：以紧凑卡片形式汇总用户当前积分、返佣、VIP、卡券四类权益的核心数据与升级进度。
- **快捷导航**：各权益卡片点击后跳转至对应子页面，不在本页承载完整业务操作。

## 0.2 页面结构

页面采用**单列纵向滚动 + 底部固定 Tab 导航**布局，自上而下依次为：

| 区域 | 说明 |
|---|---|
| 顶部导航栏 | 居中标题「奖励中心」，无左右操作按钮 |
| 活动专区 | 奖池汇总 + 热门活动推荐，合并为一张整体卡片 |
| 我的权益 | 2×2 紧凑网格，展示四类权益摘要卡片 |
| 底部导航栏 | 首页、行情、永续合约、**奖励（当前高亮）**、我的 |

- 活动专区与权益区内容可纵向滚动；底部导航栏始终固定于屏幕底部。
- 本页**不设**底部引导文案、营销 Slogan 等冗余说明区块。

## 0.3 子页面跳转关系

| 本页入口 | 跳转目标      |
|---|-----------|
| 活动专区「全部」/ 热门活动卡片 | 活动中心页     |
| 积分卡片 | 积分页       |
| 邀请返佣卡片 | 邀请返佣页     |
| VIP 卡片 | VIP 费率等级页 |
| 我的卡券卡片 | 卡券中心页     |

# 1. 顶部导航栏

## 1.1 布局

- 居中展示页面标题：**奖励中心**。
- 左右两侧为占位平衡，**不展示**「全部活动」等额外入口（活动入口统一放在活动专区内）。

## 1.2 交互

- 本页为 Tab 根页面，顶部**不提供返回按钮**。
- 用户通过底部 Tab 切换离开本页。

# 2. 活动专区

## 2.1 功能概述

将「进行中总奖池」与「热门活动推荐」合并为**一张视觉整体**的紫色渐变卡片，强化活动板块的整体感，避免奖池与活动割裂展示。

## 2.2 上半区：奖池汇总

奖池汇总区根据是否存在进行中活动，展示两种形态：

**有进行中活动时（默认）**

| 元素 | 说明 |
|---|---|
| 区块标题 | 「热门活动」 |
| HOT 标签 | 红色胶囊标签，表示当前有热门活动进行中 |
| 全部入口 | 文案「全部 →」，点击跳转活动中心 |
| 进行中总奖池 | 标签「进行中总奖池」+ 金额（如 250,000 USDT） |
| 进行中场数 | 右侧展示进行中活动数量（如「3 场进行中」） |

- 总奖池金额为当前所有进行中活动奖池的汇总值，由后台或接口聚合返回。

**无进行中活动时**

| 元素 | 说明 |
|---|---|
| HOT 标签 | **隐藏** |
| 进行中场数 | **隐藏** |
| 奖池标签 | 改为「**历史累计已瓜分奖池**」 |
| 奖池金额 | 展示平台历史活动累计已瓜分总额（如 12,850,000 USDT） |
| 辅助提示 | 可选展示「暂无进行中活动，敬请期待」 |
| 全部入口 | 保留，仍可进入活动中心查看历史 / 即将开始的活动 |

- 历史累计金额为平台维度统计，不代表用户个人收益。

## 2.3 下半区：热门活动推荐

整块下半区可点击，跳转具体活动详情页

| 元素   | 说明                           |
|------|------------------------------|
| 活动名称 | 如「ForX 交易大赛」                 |
| 参与人数 | 如「8,942 人参与」                 |
| 活动名称 | 如「合约交易争霸赛」                   |
| 本场奖池 | 标签「本场奖池」+ 金额（如 100,000 USDT） |
| 行动按钮 | 「立即参与」，视觉强调，随卡片整体可点          |

- 按奖池规模排序取 Top 1 展示；**无进行中活动时，下半区整体隐藏**，仅保留上半区奖池汇总卡片。

## 2.4 视觉要求

- 上下半区之间以细分隔线区分，但外轮廓、圆角、背景渐变保持一致，形成**一体化活动卡片**。
- 不要求在本卡片内展示活动规则长文案，详情在活动中心查看。

# 3. 我的权益

## 3.1 功能概述

以 **2×2 紧凑网格**展示用户四类权益摘要，目标是在首屏（或极少量滑动后）同时看到全部四张卡片。

- 区块标题：**我的权益**（小字号、灰色、全大写样式）。
- 每张卡片包含：模块图标 + 名称、核心指标、进度条（如有）、一行辅助说明。
- 整张卡片可点击，跳转对应子页面；右侧箭头示意可进入详情。

## 3.2 积分卡片

**跳转**：积分中心

| 展示项 | 说明 |
|---|---|
| 总积分 | 用户累计积分余额 |
| 本周新增 | 本周获得的积分增量，正向数值用强调色 |
| 加成进度 | 当前积分加成倍率（如 1.5X）、档位级别（如级别 5）、进度条 |
| 辅助说明 | 伞下用户周交易额及距下一档差额；已达最高档时展示「已达最高档」类文案 |

**加成规则说明**（摘要区展示逻辑，完整阶梯见积分中心）：

- 加成倍率根据伞下用户（含间接邀请）的**每周总交易额**自动提升。
- 进度条反映当前档位向下一档的推进情况；已达 MAX 档位时进度条满格。

## 3.3 邀请返佣卡片

**跳转**：邀请返佣

| 展示项 | 说明                                                                    |
|---|-----------------------------------------------------------------------|
| 返佣比例 | 当前返佣比例（如 10%）                                                         |
| 累计佣金 | 历史累计返佣金额，需明确标注「累计佣金」及 USDT 单位，避免用户误解为其他数值                             |
| 升级进度 | 下一返佣等级目标（如 Lv3 · 15%）、完成百分比、进度条                                       |
| 辅助说明 | 升级缺口说明，<br/>情况1：活跃好友不足，显示「还差 N 位活跃好友」<br/>情况2：交易额不足，显示「还差 N USDT 交易额」 |

## 3.4 VIP 费率卡片

**跳转**：VIP 费率等级

| 展示项 | 说明                                       |
|---|------------------------------------------|
| 卡片标题 | 直接展示当前 VIP 等级（如 **VIP 1**），而非「VIP 费率」等泛称 |
| 当前费率 | 独立一行展示 Maker / Taker 费率                  |
| 升级进度 | 下一 VIP 等级目标、完成百分比、进度条                    |
| 辅助说明 | 升级条件摘要，如 14 日交易量进度                       |

## 3.5 我的卡券卡片

**跳转**：卡券中心

| 展示项 | 说明 |
|---|---|
| 有效体验金 | 当前可用体验金张数 |
| 体验金总额 | 可用体验金折合 USDT 总额 |
| 过期预警 | 若有即将过期卡券，展示醒目提示（如「50 USDT 即将过期 · 3 天内」）；无则隐藏 |

## 3.6 卡片布局与字号

- 网格：**每行 2 张卡片**，共 4 张。
- 卡片内数值字号需适配窄屏宽度，**不得溢出卡片边界**；长数字支持换行或缩小展示。
- 卡片最小高度保持统一，内容区包含标题行、指标区、进度区（卡券可为过期提示区）。

# 4. 数据刷新与状态

## 4.1 加载时机

建议在以下时机拉取 / 刷新权益摘要数据：

- 进入奖励中心页时
- 从子页面返回时（可选）
- 用户下拉刷新时（若 APP 统一支持下拉刷新）

## 4.2 无进行中活动

当接口返回进行中活动数量为 0 时，活动专区切换为「历史累计」形态：

| 区域 | 展示规则 |
|---|---|
| 奖池标签 | 「历史累计已瓜分奖池」 |
| 奖池金额 | 平台历史累计已瓜分总额 |
| HOT 标签 | 隐藏 |
| 进行中场数 | 隐藏 |
| 热门活动推荐（下半区） | **整块隐藏** |
| 全部入口 | 保留 |

- 活动卡片外轮廓、紫色渐变背景保持不变，仅内容区收缩为上半区奖池汇总。
- 「我的权益」区不受影响，已登录用户仍正常展示四类权益摘要。

## 4.3 未登录状态

未登录用户进入本页时的展示规则：

| 区域 | 展示规则 |
|---|---|
| 活动专区 | 可展示公开活动信息（有进行中活动时展示完整活动卡片；无活动时同 4.2） |
| 我的权益标题 | 保留 |
| 登录引导条 | 权益区顶部展示引导文案，如「登录后查看我的权益」+ **登录** 按钮 |
| 权益卡片数值 | 核心指标展示 `--` 占位，不展示真实用户数据 |
| 进度条 / 费率 / 过期预警 | 隐藏 |
| 卡片辅助文案 | 可展示「登录查看」 |
| 卡片点击 | 拦截跳转子页面，触发登录流程（与 APP 全局登录策略一致） |

- 未登录状态下不请求用户权益类接口，或接口返回空时按本规则渲染。

## 4.4 网络异常

当活动或权益数据接口请求失败（超时、断网等）时，页面在**未登录占位态基础上**展示全页 `--` 数据，并在活动专区上方增加红色异常提示条。

| 区域 | 展示规则 |
|---|---|
| 顶部导航栏 | 正常展示 |
| 底部 Tab 导航 | 正常展示，不影响切换 |
| **异常提示条** | 活动专区**上方**展示红色横条，文案：「**加载失败，请检查网络后重试**」；右侧可提供「重试」入口 |
| 活动专区 | 保留卡片结构与布局；奖池金额、进行中场次、参与人数、活动名称、本场奖池等**全部展示 `--`**；HOT 标签隐藏 |
| 我的权益 | 同未登录态：数值 `--`、隐藏进度条 / 费率 / 过期预警；**不展示**登录引导条 |
| 卡片点击 | 拦截跳转子页面 |

- 网络异常态与未登录态的数据占位规则一致，差异在于：网络异常有顶部红色提示条，且无登录引导。
- 用户点击「重试」后重新发起数据请求；请求成功则恢复默认展示。

## 4.5 其他空状态

| 场景 | 处理建议 |
|---|---|
| 无可用卡券 | 卡券卡片数量展示 `--`，隐藏过期预警 |
| 权益单项无数据 | 对应字段展示 `--` 或 0，不展示进度条 |
