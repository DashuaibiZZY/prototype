# APP 端合约交易页

# **【重要说明】**

> 本文档为 APP 端合约交易页面的交互与功能需求说明，供产品、UI 及前端开发参考。

> 本系统基于合作方成熟的合约交易系统进行前端对接开发。文档中涉及的**核心交易规则、风控计算逻辑、保证金公式、强平机制、资金费率算法、订单校验与生命周期等底层业务逻辑，均以对方产研团队提供的规则为准**。本文档仅描述 APP 端页面的信息展示、用户操作流程及交互行为，如文档描述与对方系统实际规则存在差异，以对方系统实际规则为准。

> 功能规则定义与 Web 端《合约交易页》保持一致；本文档重点描述 **APP 移动端布局、底部弹层、卡片式列表、页面跳转** 等与 Web 不同的交互形态。

> 如有涉及功能层面不一致的问题，以产研团队的修改建议为准。

# 0\. 页面概述与整体布局

## 0\.1 功能定位

APP 合约交易页是用户进行永续合约开仓、平仓、查看盘口与管理持仓/委托的核心页面。与 Web 端不同，本页**不内嵌 K 线图表区域**；用户可通过顶部入口跳转至「行情详情页」查看完整 K 线、深度及资金费率。

## 0\.2 页面结构

页面采用**单列纵向滚动 \+ 底部固定 Tab 导航**布局，自上而下依次为：

|区域|说明|
|---|---|
|顶部导航栏|交易对切换、涨跌幅、K 线入口、更多功能|
|中部交易区|左侧盘口深度（约 40% 宽）\+ 右侧交易终端（约 60% 宽）|
|仓位管理区|当前持仓 / 当前委托 Tab，卡片列表|
|底部导航栏|首页、行情、永续合约（当前高亮）、奖励、我的|

- 中部及仓位管理区内容可纵向滚动；底部导航栏始终固定于屏幕底部。

- 各类设置、筛选、确认操作以**底部弹层**或**居中弹窗**形式呈现，覆盖于页面内容之上。

## 0\.3 与 Web 端的主要差异

|能力|Web 端|APP 端|
|---|---|---|
|K 线 / 深度图 / 合约详情 / 资金费率视图|内嵌于图表区四 Tab|K 线、资金费率等跳转行情详情页；盘口内嵌于左侧；交易对详情/风险限额跳转独立子页|
|市场信息栏|顶部横向多指标|精简为：交易对名 \+ 24h 涨跌幅徽章；资金费率在盘口顶部|
|偏好设置|导航栏右侧抽屉|更多菜单 → 交易偏好设置页|
|交易终端|页面右侧固定面板|与盘口左右分栏，位于屏幕中部右侧|
|持仓/委托|表格|卡片列表|
|历史流水|内嵌底部数据 Tab|跳转「交易流水页面」|

# 1\. 顶部导航栏

## 1\.1 功能概述

展示当前交易对标识与 24h 涨跌幅，提供交易对切换、K 线详情入口及更多功能菜单。

## 1\.2 交易对与涨跌幅

**（1）交易对名称**

- 左侧展示当前交易对名称（如 BTCUSDT），字体加粗。

- 名称右侧带下拉箭头，表示可点击切换。

- 点击整块区域，从底部弹出「选择合约」弹层（见 6\.2）。

**（2）24h 涨跌幅徽章**

- 紧邻交易对名称右侧，胶囊形徽章展示 24h 涨跌幅百分比（如「+2.45%」）。

- 涨（≥0）背景绿色、文字白色；跌（\<0）背景红色；平盘或数据缺失时灰色。

- 数值带正负号，保留两位小数。

## 1\.3 K 线入口

- 右侧第一个图标按钮，点击跳转至当前交易对的**行情详情页**，查看 K 线、订单簿、最新成交、资金费率等完整行情信息。

- 跳转时携带当前交易对上下文。

## 1\.4 更多功能菜单

- 右侧第二个图标按钮（「…」），点击从底部弹出「更多功能」弹层（见 6\.9）。

- 入口包含：偏好设置、充值、资金费率、交易对信息（交易对详情页）、风险限额、添加/取消自选等。

- 打开弹层前刷新当前交易对的自选状态展示。

# 2\. 盘口深度区

盘口深度区位于中部交易区**左侧**，约占屏幕宽度 40%，与右侧交易终端并排展示。

## 2\.1 功能概述

以紧凑纵向布局展示当前交易对的实时买卖挂单深度、最新价、标记价及资金费率倒计时，支持精度切换与深度可视化。

## 2\.2 资金费率 / 倒计时

- 位于盘口区域最上方，文案格式：「資金費率 / 时:分:秒」（距下次结算倒计时）。

- 带虚线下划线，点击从底部弹出「资金费率」说明弹层（见 6\.8）。

- 弹层内展示：即时资金费率、结算周期、下次结算倒计时、年化利率及说明文案；底部提供「查看资金费率历史」跳转入口。

- 倒计时每秒更新。

## 2\.3 表头

- 两列：「價格(USDT)」与「數量（单位）」

- 数量列单位随交易终端「货币单位」切换同步变更（如 BTC / USDT）。

## 2\.4 卖单区（Asks）

- 位于盘口上半部分，卖方挂单，价格**从高到低**排列（靠近中间最新价一侧价格较低）。

- 每行：价格（红色）、数量。

- 每行背景有**红色深度渐变条**，宽度表示该档挂单量相对卖盘最大档位的占比；数据变化时宽度过渡动画更新（约 0\.55s）。

- 卖盘深度条自右向左延伸。

## 2\.5 中间价格区

- 居中展示：

    - **最新成交价**：较大字号，涨绿跌红。

    - **标记价格**：较小灰色字号，带虚线下划线；点击可展示说明（标记价格用于计算盈亏与强平）。

- 两价随行情实时更新。

## 2\.6 买单区（Bids）

- 位于盘口下半部分，买方挂单，价格**从高到低**排列。

- 每行：价格（绿色）、数量。

- 每行背景有**绿色深度渐变条**，宽度表示买盘深度占比；自右向左延伸。

## 2\.7 盘口精度切换

- 默认显示当前生效的精度值（如“0\.01”），旁边有下拉三角图标。

- 点击展开下拉菜单，菜单内提供 **4 个可选精度档位**，具体档位由当前交易对的基础价格精度动态决定：

    - 第一档：基础精度（即该交易对的最小价格变动单位，由后台配置，如“0\.01”）。

    - 第二档：基础精度 × 10（如“0\.1”）。

    - 第三档：基础精度 × 100（如“1”）。

    - 第四档：基础精度 × 1000（如“10”）。

- 每个档位的显示文本即为计算后的精度数值（如“0\.01”“0\.1”“1”“10”）。

- 用户选择任意档位后，订单簿内的挂单价格将按该精度进行聚合显示，同时订单簿重新渲染。

- 下拉菜单采用绝对定位，点击外部自动关闭。

- 若交易对价格精度发生变更（例如切换交易对），精度档位自动重新计算，并重置为第一档（基础精度）。

## 2\.8 数据更新

- 盘口数据通过长连接推送或定时轮询实时更新；仅刷新变化行，避免整表闪烁。

- 数量大额使用适当缩写（K/M/B）。

## 2\.9 新币未开盘状态（盘口区）

当选中尚未正式开启交易的合约时，正常盘口内容隐藏，盘口区域被**待开盘倒计时面板**覆盖：

**（1）展示内容**

- 币种图标与「新幣上線」标签。

- 交易对名称（如 PEPE / USDT）。

- 文案：「即將開啟交易」。

- 倒计时：天 : 时 : 分 : 秒，每秒递减。

**（2）联动规则**

- 顶部 24h 涨跌幅徽章显示 「--」，灰色样式。

- 资金费率倒计时显示 「--」。

- 交易终端「最大可開」「強平價格」等字段显示 「--」（见 8\.2）。

- 盘口深度条停止刷新。

# 3\. 交易终端

交易终端位于中部交易区**右侧**，是用户执行下单操作的核心区域。业务规则与 Web 端交易终端一致，交互以 APP 底部弹层、内联选择器为主。

## 3\.1 功能概述

支持开仓/平仓切换、限价/市价/止盈止损下单、保证金模式与杠杆调整、最优价（对手价/同向价）、跟随止盈止损、数量单位切换及下单二次确认。

## 3\.2 仓位模式与杠杆

### 3\.2\.1 仓位模式切换

**业务规则（与 Web 一致）：**

- 每个交易对的全仓/逐仓独立设置；首次默认全仓。

- 存在未平仓持仓时**不允许切换**，确认时提示：「当前合约存在持仓，无法切换保证金模式。」

- 底部说明：「调整保证金模式仅对当前合约生效。」

**APP 交互：**

- 交易终端顶部左侧按钮显示当前模式（如「全倉」）。

- 点击从底部弹出「保證金模式」弹层：全仓 / 逐仓两个可选卡片，选中高亮；底部「確認」保存并关闭。

### 3\.2\.2 杠杆倍数设置

**业务规则（与 Web 一致）：**

- 做多杠杆、做空杠杆**独立**设置，显示为「20x / 10x」（绿/红分色）。

- 杠杆档位由后台杠杆阶梯配置，非任意输入。

- 有持仓时对应方向杠杆**只允许调高**，调低提示：「当前合约存在持仓，无法调低杠杆。请先平仓后再操作。」

- 可开名义价值随杠杆阶梯联动计算。

**APP 交互：**

- 点击杠杆按钮，从底部弹出「調整槓桿」弹层（可滚动，最高约 85% 屏高）。

- 弹层分「做多杠杆」「做空杠杆」两个区域，各含：当前倍数预览、步进器（− / 数值 / \+）、滑块、调整后最大可开提示。

- 底部「確認」保存，并同步刷新：终端杠杆按钮、持仓卡片杠杆标签、委托卡片等。

- 持仓卡片内「槓桿」按钮可快捷打开同一弹层。

## 3\.3 开仓/平仓切换

**（1）切换控件**

- 灰底分段控件：「開倉」（默认白底阴影高亮）、「平倉」（灰色）。

- 互斥切换。

**（2）联动规则**

- 切换至**平仓**时：隐藏「跟随止盈止损」勾选区域；显示平仓按钮区（可平多/可平空数量 \+ 平多/平空按钮）。

- 切换至**开仓**时：恢复跟随止盈止损区域；显示开多/开空按钮及最大可开、强平价格提示。

- 订单类型为「止盈止损」时，触发价输入区随开/平仓模式变化（见 3\.5\.3）。

**（3）下单按钮（开仓模式）**

- 绿色「買入 / 做多」：开多方向。

- 红色「賣出 / 做空」：开空方向。

- 各自上方展示：最大可開、強平價格（该方向）。

**（4）下单按钮（平仓模式）**

- 展示可平多数量 \+ 红色「平多」按钮。

- 展示可平空数量 \+ 绿色「平空」按钮；无可平空仓时按钮置灰禁用。

- 点击平多/平空进入下单确认流程（交易终端内）；**持仓卡片「平倉」按钮**则先打开平仓设置弹层（见 4\.3\.3）。

## 3\.4 订单类型选择

**（1）触发方式**

- 订单类型行：左侧信息图标 \+ 当前类型文案（如「限價委託」）\+ 右侧箭头。

- 点击从底部弹出「訂單類型」弹层。

**（2）可选项**

|选项|说明|
|---|---|
|限價委託|指定价格挂单|
|市價委託|按市场最优价成交|
|止盈止損|条件触发单（独立订单类型）|

- 选中后更新订单类型文案，关闭弹层，并联动价格区、止盈止损区显示/隐藏。

- 弹层底部提供「取消」按钮。

## 3\.5 订单类型与下单参数

### 3\.5\.1 限价单

**参数（与 Web 一致）：**

|字段|说明|
|---|---|
|價格|可编辑，默认填入最新价；单位 USDT|
|數量|手动输入 \+ 百分比滑杆（0%/25%/50%/75%/100%）|
|货币单位|BTC / USDT 切换，切换后清空并按规则换算|
|可用余额|展示可用 USDT；右侧「\+」进入充币/划转（见 3\.10）|
|止盈/止损（可选）|跟随止盈止损勾选（仅开仓模式，详见 3\.7）|

**价格区交互：**

- 价格输入框上方浮层标签「價格」。

- 右侧「最优价」按钮（见 3\.6）：开启后价格区变为最优价选项展示，点击打开最优价选择弹层。

**数量区交互：**

- 输入框右侧内嵌货币单位选择器，点击弹出「货币单位」底部列表（BTC / USDT）。

- 下方 5 档圆点百分比滑杆（视觉刻度 0/25/50/75/100%），拖动按可用余额比例填充数量。

**校验（与 Web 一致）：**

- 价格：数值、\>0、符合价格精度。

- 数量：符合精度、≤ 最大可开、≥ 最小交易单位。

### 3\.5\.2 市价单

**参数：**

|字段|说明|
|---|---|
|價格|固定显示「市價」，不可编辑；最优价按钮不可用|
|數量|同限价单|
|止盈/止损（可选）|跟随止盈止损勾选（仅开仓模式，详见 3\.7）|

- 订单有效期后端为 IOC（与 Web 一致）。

### 3\.5\.3 止盈止损单

**功能定位（与 Web 5\.4\.3 一致）：**

- 条件订单：预设触发价，达标后自动提交开仓/平仓委托。

- 与限价/市价上的「跟随止盈止损」勾选框是**不同功能**（跟随止盈止损见 3\.7）。

**（1）委托模式**

- 顶部内联选择器，默认「单向」；点击打开通用选择器弹层：

    - **单向**：预设触发价与委托价；市价达触发价后按委托价下单。

    - **双向**：预设止盈/止损触发价与委托价；一侧触发成交后另一侧自动失效。

**（2）输入区结构（动态渲染）**

每个触发条件行包含：

|元素|说明|
|---|---|
|标签|开仓双向：两个「触发价格」；开仓单向：一个「触发价格」；平仓双向：「止盈」「止损」；平仓单向：「触发价格」|
|触发价输入|数字输入框|
|触发价类型|内联选择器，默认「最新价格」；可选「标记价格」（APP 原型）；Web 另支持指数价格|
|委托类型|内联选择器，文案如「市價委託」「限價委託」「止盈市價」等，随开/平仓及双向/单向变化|
|委托价输入|选限价时显示；选市价时隐藏|

- 委托类型、触发价类型均通过**通用选择器弹层**选择，确保叠置于交易相关弹层之上。

**（3）方向与按钮**

- **开仓**：用户点击「買入/做多」或「賣出/做空」提交，方向由按钮决定。

- **平仓**：根据持仓方向显示平多/平空；止盈止损判定规则与 Web 一致（触发价与开仓均价关系）。

**（4）验证规则**

**触发价验证：**

|场景|验证规则|
|---|---|
|开仓双向 \+ 开多|两个触发价均须 \> 当前最新成交价|
|开仓双向 \+ 开空|两个触发价均须 \< 当前最新成交价|
|开仓单向 \+ 开多|触发价须 \> 当前最新成交价|
|开仓单向 \+ 开空|触发价须 \< 当前最新成交价|
|平仓双向（持多）|止盈触发价 \> 持仓开仓均价；止损触发价 \< 持仓开仓均价|
|平仓双向（持空）|止盈触发价 \< 持仓开仓均价；止损触发价 \> 持仓开仓均价|
|平仓单向（持多/持空）|触发价须 ≠ 持仓开仓均价|

- 所有触发价不能等于当前最新成交价。

- 触发价小数精度须符合交易对价格精度。

**委托价验证（仅限价委托时）：**

|方向|验证规则|
|---|---|
|开多|委托价 ≥ 触发价|
|开空|委托价 ≤ 触发价|
|平多|委托价 ≥ 触发价|
|平空|委托价 ≤ 触发价|

**数量验证：**

- 与限价单一致（精度、最大可开、最小交易单位）；开仓须检查保证金充足。

**（5）委托类型文案（APP 展示）**

内联选择器根据场景显示不同文案，便于用户识别：

|场景|市价|限价|
|---|---|---|
|开仓|市價委託|限價委託|
|平仓双向·止盈|止盈市價|止盈限價|
|平仓双向·止损|止損市價|止損限價|
|平仓单向|平倉市價|平倉限價|

**（6）生命周期**

- 未触发：当前委托 → 止盈止损单子 Tab；可撤单、修改。

- 已触发：历史委托；双向单一边成交后另一边「已取消」。

### 3\.5\.5 订单有效期

与 Web 端 5\.6 一致：

|类型|限价单|市价单|止盈止损单|
|---|---|---|---|

## 3\.7 跟随止盈止损（勾选框）

**（1）功能定位**

- 该功能为**附加在限价单/市价单主订单上的条件单**，非独立订单类型。

- 仅在**开仓模式**且订单类型为**限价/市价**时显示；平仓模式下完全隐藏。

- 用户为主订单预设止盈/止损触发价；主订单**完全成交**后，系统按预设触发价自动挂出**市价止盈单**与**市价止损单**（各一张）。

    

**（2）展示与交互**

- 勾选「止盈/止損」复选框，展开止盈/止损输入区；取消勾选则收起。

- 文字标签「止盈/止損」，标签下方有虚线下划线。

- APP 点击标签展示说明文案：

\> 「止盈止損是指下單時提前為該訂單預設止盈價及止損價，以鎖定利潤或控制損失。當訂單完全成交時，系統將自動進行止盈止損委託。」

    

**（3）输入区结构**

- **止盈区域**（上排）：止盈触发价格输入框 \+ 触发价格类型下拉（默认「最新成交價」，可选「標記價格」）。

- **止损区域**（下排）：止损触发价格输入框 \+ 触发价格类型下拉（默认同上）。

- 各触发价输入框下方可显示**预估收益**行（见（4））；两组输入区下方可显示**冲突提示**行。

    

**（4）预估收益计算**

跟随止盈止损的预估收益，是在用户填写**主单数量**与**止盈/止损触发价**后，按主单**委托价**估算该腿触发后的盈亏金额。计算与展示分四步完成。

    

**第一步：确定委托价 \(P_{ref}\)**

预估收益统一以主订单的**委托价**为基准，记为 \(P_{ref}\)：

|主订单类型|\(P_{ref}\) 取值|说明|
|---|---|---|
|限价单|用户输入的**委托价**（限价输入框数值）|随用户修改限价实时更新|
|市价单|当前合约**最新成交价**|价格区显示「市價」不可编辑时，取盘口/行情区展示的最新成交价|

    

**市价单委托价刷新：**

- 主订单为**市价单**时，\(P_{ref}\) 取自最新成交價，该价格会随行情变动。

- 预估收益以 **500ms** 为周期自动重算并刷新展示（数量、触发价不变时，仅因最新成交價变化而更新）。

- 用户修改数量或止盈/止损触发价时，仍**立即**触发重算，不等待下一个 500ms 周期。

    

**第二步：换算数量 \(Q\)（USDT 名义价值）**

|数量单位|换算方式|
|---|---|
|USDT|直接取数量输入值|
|BTC（等基础资产）|\(Q =\) 数量 × \(P_{ref}\)|

数量或 \(P_{ref}\) 无效时，不展示预估。

    

**第三步：推断整组方向（做多 / 做空 / 冲突）**

一组止盈/止损在同一时刻只能对应**一个方向**。系统根据已填写的触发价与 \(P_{ref}\) 比较，推断整组方向：

|已填字段|推断为**做多**|推断为**做空**|推断为**冲突**|
|---|---|---|---|
|止盈 + 止损均已填|止盈触发价 \> \(P_{ref}\) **且** 止损触发价 \< \(P_{ref}\)|止盈触发价 \< \(P_{ref}\) **且** 止损触发价 \> \(P_{ref}\)|其余组合（如两价同在 \(P_{ref}\) 上方或下方）|
|仅填止盈|止盈触发价 \> \(P_{ref}\)|止盈触发价 \< \(P_{ref}\)|—|
|仅填止损|止损触发价 \< \(P_{ref}\)|止损触发价 \> \(P_{ref}\)|—|
|均未填|—|—|不计算|

**冲突时**：隐藏全部预估收益行，并展示冲突提示（见（6））。

    

**第四步：计算各腿金额并组装展示文案**

对**止盈**、**止损**各腿分别判断；两腿独立计算，满足条件才显示对应行。

**单腿展示条件（须同时满足）：**

1. 主单**数量**已输入有效数值；

2. 该腿**触发价**已输入有效数值；

3. 整组方向**不是冲突**；

4. 该腿的「腿方向」与整组方向**一致**（见下表）。

    

**单腿方向判定（该腿触发价 vs \(P_{ref}\)）：**

|腿|腿方向为**做多**|腿方向为**做空**|
|---|---|---|
|止盈|触发价 \> \(P_{ref}\)|触发价 \< \(P_{ref}\)|
|止损|触发价 \< \(P_{ref}\)|触发价 \> \(P_{ref}\)|

仅当「腿方向 = 整组方向」时，才展示该腿预估。例如整组推断为做多时，止盈触发价须高于 \(P_{ref}\)、止损触发价须低于 \(P_{ref}\)，两腿方向才均为做多，两行预估才会同时出现。

    

**金额计算公式（\(Q\) 以 USDT 计，\(P_{trigger}\) 为该腿触发价）：**

- **做多**：预估收益 \(= Q \times (P_{trigger} - P_{ref}) / P_{ref}\)

- **做空**：预估收益 \(= Q \times (P_{ref} - P_{trigger}) / P_{ref}\)

计算时采用整组推断出的方向（做多或做空）选用对应公式。

    

**展示文案组装规则：**

文案由三段拼接：`{整组方向}{腿类型}預估收益 {带符号金额} USDT`

|片段|取值规则|
|---|---|
|整组方向|第三步推断结果：做多 → `做多`；做空 → `做空`|
|腿类型|止盈腿 → `止盈`；止损腿 → `止損`|
|带符号金额|收益 ≥ 0 前缀 `+`，收益 \< 0 前缀 `-`；保留两位小数，单位 USDT|

**颜色规则：**

- 金额 ≥ 0：绿色字体

- 金额 \< 0：红色字体

    

**示例（\(P_{ref} = 70{,}000\)，数量 \(Q = 1{,}000\) USDT）：**

|场景|止盈触发价|止损触发价|整组方向|止盈行展示|止损行展示|
|---|---|---|---|---|---|
|做多双填|75,000|65,000|做多|`做多止盈預估收益 +71.43 USDT`（绿）|`做多止損預估收益 -71.43 USDT`（红）|
|做空双填|65,000|75,000|做空|`做空止盈預估收益 +71.43 USDT`（绿）|`做空止損預估收益 -71.43 USDT`（红）|
|仅填止盈（做多）|75,000|—|做多|`做多止盈預估收益 +71.43 USDT`（绿）|不展示|
|仅填止盈（做空）|65,000|—|做空|`做空止盈預估收益 +71.43 USDT`（绿）|不展示|
|方向冲突|75,000|72,000|冲突|不展示（显示冲突提示）|不展示|

**刷新触发规则：**

- **限价单**：委托价、数量、触发价任一变化时，立即重算并刷新展示。

- **市价单**：数量、触发价变化时立即重算；另每 **500ms** 按最新成交價自动刷新一次预估收益。

    

**（5）触发逻辑**

- **完全成交**：主订单完全成交后，系统根据已设置的止盈/止损触发价，自动挂出**两张市价条件单**——一张**市价止盈**、一张**市价止损**。

- **未成交 / 部分成交**：止盈/止损条件单**不激活**；用户可在**当前挂单**上修改止盈/止损触发价格（直至主订单完全成交或取消）。

- **主单取消**：主订单被取消后，跟随的止盈/止损条件单**不激活**、不挂出。

    

**（6）验证规则**

**价格相对关系（相对主订单入场/委托价）：**

- **做多**：止盈触发价 \> 入场价；止损触发价 \< 入场价。

- **做空**：止盈触发价 \< 入场价；止损触发价 \> 入场价。

    

**方向冲突提示：**

- 止盈、止损触发价**同时高于**委托价：提示 `止盈止損價不能同時高於委託價`。

- 止盈、止损触发价**同时低于**委托价：提示 `止盈止損價不能同時低於委託價`。

- 发生冲突时，不展示预估收益行。

    

**其余规则：**

- 触发价须为有效正数，精度与交易对价格精度一致。

- 其余验证规则与止盈止损订单类型一致（见 3\.5\.3）。

# 6\. 底部弹层与居中弹窗

## 6\.1 层级规范

弹层与弹窗按叠放顺序由低到高分为五层，后打开的须覆盖先打开的同级内容；嵌套选择时，**通用选择器弹层**必须高于**持仓相关底部弹层**，避免选项被遮挡。

|叠放顺序|适用场景|
|---|---|
|第一层：普通底部弹层|订单类型、杠杆调整、保证金模式、交易对选择、货币单位、盘口精度、资金费率说明、更多功能、最优价等常规设置|
|第二层：持仓相关底部弹层|持仓止盈止损设置、持仓平仓设置|
|第三层：居中确认弹窗|下单确认、一键平仓确认、持仓止盈止损二次确认|
|第四层：通用选择器弹层|触发价类型、止盈止损委托模式、订单委托类型、平仓价格类型等嵌套选项|
|第五层：委托筛选面板|当前委托列表的合约筛选、订单类型筛选|

**补充说明：**

- 同一层级内，后触发的弹层覆盖先触发的弹层。

- 从持仓相关弹层内唤起通用选择器时，选择器须完整展示在最上层，关闭选择器后回到原弹层。

- 委托筛选面板打开时覆盖页面主体，但不影响底部导航栏的固定展示（若产品全局如此设计）。

## 6\.2 选择合约弹层（交易对切换）

**结构与 Web 2\.3 业务规则一致，交互为底部弹层：**

**（1）结构**

**（2）交互**

- 点击遮罩关闭。

**（2）分类 Tab 行为**

# 7\. 关联子页面

以下三个子页面由合约交易页「更多功能」菜单或交易对详情页内链进入，均为独立全屏页面（非弹层）。业务字段与 Web 端合约详情视图（Web 文档 §3\.5）保持一致。

## 7\.1 交易偏好设置页

### 7\.1\.1 功能概述

集中管理合约下单二次确认与成交提示音等个人偏好。对应 Web 端导航栏「偏好设置抽屉」，APP 以独立子页面呈现。

### 7\.1\.2 入口与返回

- **入口**：合约交易页 → 更多功能弹层 →「偏好设置」。

- **返回**：顶部左侧返回按钮，回到合约交易页；应尽量保持离开前的交易对及开平仓上下文。

### 7\.1\.3 页面结构

**（1）顶部导航栏**

- 左侧：返回按钮。

- 居中标题：「偏好設置」。

**（2）下单确认**

分组标题「下單確認」，包含三个开关项，控制对应类型订单提交前是否弹出二次确认弹窗：

|开关项|默认状态|说明|
|---|---|---|
|限价 订单|开启|关闭后，限价单点击下单直接提交|
|市价 订单|开启|关闭后，市价单点击下单直接提交|
|止盈止損 订单|开启|关闭后，止盈止损单点击下单直接提交|

- 每项为 Toggle 开关样式，用户可独立配置。

- 规则与 Web 端 §1\.1 一致。

**（3）消息通知**

分组标题「消息通知」：

|开关项|默认状态|说明|
|---|---|---|
|成交提示音|开启|控制下单成功、失败等事件是否播放提示音|

**（4）底部操作**

- 页面底部固定「恢复至默认设置」文字按钮。

- 点击后将上述所有开关恢复为系统默认值，并即时保存。

### 7\.1\.4 数据持久化

- 偏好设置保存至后端或本地存储，下次登录保持用户上次状态。

- 在偏好设置页修改后，返回合约交易页立即生效。

## 7\.2 交易对详情页

### 7\.2\.1 功能概述

展示当前交易对的合约规格与交易规则参数，帮助用户了解该合约的下单限制与风控参数。对应 Web 端图表区「合约详情」视图。

### 7\.2\.2 入口与参数

- **入口**：合约交易页 → 更多功能弹层 →「交易对信息」。

- **携带参数**：当前交易对名称（如 BTCUSDT）；页面所有字段按该交易对加载。

- **返回**：顶部左侧返回按钮，回到合约交易页。

### 7\.2\.3 页面结构

**（1）顶部导航栏**

- 左侧：返回按钮。

- 居中标题：「交易對信息」。

**（2）标题区**

- 主标题：「{交易对} 永續」（如 BTCUSDT 永續），随入参动态展示。

- 右侧标签：「交易規則」（展示用标签，可点击跳转完整规则文档，若产品提供）。

**（3）基础信息区**

以两列网格展示以下字段（字段值由后台合约配置下发）：

|字段|说明|
|---|---|
|市場|合约名称（如 BTCUSDT）|
|最大槓桿|该合约支持的最大杠杆倍数（如 100x）|
|最小價格增量|价格最小变动单位（如 $0.1）|
|限價單單筆最大訂單數量|单笔限价单允许的最大下单量|
|最小交易單位|最小可交易数量|
|市價單單筆最大訂單數量|单笔市价单允许的最大下单量|
|清算費率|强平清算时收取的费用比例|
|最大倉位|单账户在该合约可持有的最大仓位|

- 数值带单位时须展示完整单位（如 BTC、USDT、%）。

- 数据加载中可显示占位符；加载失败展示错误提示并支持重试。

**（4）风险限额入口**

- 信息区底部展示「查看風險限額」行，右侧箭头。

- 点击跳转**交易对风险限额页**（见 7\.3），携带同一交易对参数。

### 7\.2\.4 数据更新

- 切换交易对后从详情页进入时，整页按新交易对刷新。

- 合约参数变更时，用户下次进入页面拉取最新配置。

## 7\.3 交易对风险限额页

### 7\.3\.1 功能概述

以表格形式展示当前交易对的仓位价值阶梯与对应保证金率、可用杠杆，供用户了解不同仓位规模下的风控要求。对应 Web 端合约详情视图中的「风险限额表格」。

### 7\.3\.2 入口与参数

- **入口一**：合约交易页 → 更多功能弹层 →「風險限額」。

- **入口二**：交易对详情页 →「查看風險限額」。

- **携带参数**：当前交易对名称；表格数据按该交易对加载。

- **返回**：顶部左侧返回按钮，返回上一页（合约交易页或交易对详情页）。

### 7\.3\.3 页面结构

**（1）顶部导航栏**

- 左侧：返回按钮。

- 居中标题：「風險限額」。

**（2）副标题区**

- 上行小字：「{交易对} 永續」。

- 下行标题：「風險限額 (Risk Limits)」。

**（3）风险限额表格**

|列名|说明|
|---|---|
|倉位價值 (USDT)|该档位允许的仓位名义价值区间|
|維持保證金率|该区间对应的维持保证金比例|
|起始保證金率|该区间对应的起始保证金比例|
|槓桿|该区间允许的最大杠杆倍数|

**展示规则：**

- 按仓位价值从低到高排列；最后一行区间为「上限值 - ∞」，表示无上限档位。

- 当前用户所处档位可在对应行高亮（若产品支持）。

- 表格内容只读，由平台统一配置，用户不可编辑。

- 小屏下表格支持横向滑动查看全部列。

**（4）空态与异常**

- 无配置数据时展示「暫無數據」。

- 加载失败展示错误提示，支持点击重试。

# 10\. 附录：预估强平价格与保证金展示规则

与 Web 端 5\.10 / 5\.8 一致，APP 在交易终端摘要、下单确认弹窗中展示：

**（1）预估强平价格**

- 依据杠杆、开仓价（或市价）、保证金等计算；公式以对方系统为准。

- 做多且杠杆 ≤ 1 时显示 「--」。

- 价格/数量输入不完整时显示 「--」。

- 新币未开盘状态显示 「--」。

**（2）保证金**

- 开仓确认弹窗展示该笔预计占用保证金（USDT）。

- 平仓确认不展示保证金字段。

**（3）可平数量**

- 平仓模式展示「可平多」「可平空」，随持仓实时更新；无可平空仓时平空按钮禁用。

**（4）费率提示**

- 若产品需要展示吃单/挂单费率，可在交易终端底部以小字展示（与 Web 5\.10 费率行一致）。
