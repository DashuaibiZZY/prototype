# 0\. 页面概述与整体布局

## 0\.1 功能定位

APP 合约交易页是用户进行永续合约开仓、平仓、查看盘口与管理持仓/委托的核心页面。与 Web 端不同，本页**不内嵌 K 线图表区域**；用户可通过顶部入口跳转至「行情详情页」查看完整 K 线、深度及资金费率。



## 0\.2 页面结构

页面采用**单列纵向滚动 + 底部固定 Tab 导航**布局，自上而下依次为：

|区域|说明|
|---|---|
|顶部导航栏|交易对切换、涨跌幅、K 线入口、更多功能|
|中部交易区|左侧盘口深度（约 45% 宽）+ 右侧交易终端（约 55% 宽）|
|仓位管理区|当前持仓 / 当前委托 Tab，卡片列表|
|底部导航栏|首页、行情、永续合约（当前高亮）、奖励、我的|

- 中部及仓位管理区内容可纵向滚动；底部导航栏始终固定于屏幕底部。

- 各类设置、筛选、确认操作以**底部弹层（Bottom Sheet）**或**居中弹窗（Modal）**形式呈现，覆盖于页面内容之上。

    

## 0\.3 与 Web 端的主要差异

|能力|Web 端|APP 端|
|---|---|---|
|K 线 / 深度图 / 合约详情 / 资金费率视图|内嵌于图表区四 Tab|K 线、资金费率等跳转行情详情页；盘口内嵌于左侧；合约详情/风险限额跳转独立子页|
|市场信息栏|顶部横向多指标|精简为：交易对名 + 24h 涨跌幅徽章；资金费率在盘口顶部|
|偏好设置|导航栏右侧抽屉|更多菜单 → 合约偏好设置子页|
|交易终端|页面右侧固定面板|与盘口左右分栏，位于屏幕中部右侧|
|持仓/委托|表格|卡片列表|
|历史流水|内嵌底部数据 Tab|跳转「交易流水页面」|

# 1\. 顶部导航栏

## 1\.1 功能概述

展示当前交易对标识与 24h 涨跌幅，提供交易对切换、K 线详情入口及更多功能菜单。



## 1\.2 交易对与涨跌幅

**（1）交易对名称**

- 左侧展示当前交易对名称（如 BTCUSDT），字体加粗。

- 名称右侧带下拉箭头，表示可点击切换。

- 点击整块区域，从底部弹出「选择合约」弹层（见 6\.2）。

    

**（2）24h 涨跌幅徽章**

- 紧邻交易对名称右侧，胶囊形徽章展示 24h 涨跌幅百分比（如 `+2.45%`）。

- 涨（≥0）背景绿色、文字白色；跌（\<0）背景红色；平盘或数据缺失时灰色。

- 数值带正负号，保留两位小数。

    

## 1\.3 K 线入口

- 右侧第一个图标按钮，点击跳转至当前交易对的**行情详情页**（`行情详情.html`），查看 K 线、订单簿、最新成交、资金费率等完整行情信息。

- 跳转时携带当前交易对上下文。

    

## 1\.4 更多功能菜单

- 右侧第二个图标按钮（「…」），点击从底部弹出「更多功能」弹层（见 6\.9）。

- 入口包含：偏好设置、充值、资金费率、交易对信息、风险限额、添加/取消自选等。

- 打开弹层前刷新当前交易对的自选状态展示。

# 2\. 盘口深度区

盘口深度区位于中部交易区**左侧**，约占屏幕宽度 45%，与右侧交易终端并排展示。



## 2\.1 功能概述

以紧凑纵向布局展示当前交易对的实时买卖挂单深度、最新价、标记价及资金费率倒计时，支持精度切换与深度可视化。



## 2\.2 资金费率 / 倒计时

- 位于盘口区域最上方，文案格式：`資金費率 / HH:MM:SS`（距下次结算倒计时）。

- 带虚线下划线，点击从底部弹出「资金费率」说明弹层（见 6\.8）。

- 弹层内展示：即时资金费率、结算周期、下次结算倒计时、年化利率及说明文案；底部提供「查看资金费率历史」跳转入口。

- 倒计时每秒更新。

    

## 2\.3 表头

- 两列：`價格(USDT)` \| `數量({单位})`

- 数量列单位随交易终端「货币单位」切换同步变更（如 BTC / USDT）。

    

## 2\.4 卖单区（Asks）

- 位于盘口上半部分，卖方挂单，价格**从高到低**排列（靠近中间最新价一侧价格较低）。

- 每行：价格（红色）、数量。

- 每行背景有**红色深度渐变条**，宽度表示该档挂单量相对卖盘最大档位的占比；数据变化时宽度过渡动画更新（约 0\.55s）。

- 卖盘深度条自右向左延伸。

    

## 2\.5 中间价格区

- 居中展示：

    - **最新成交价**：较大字号，涨绿跌红。

    - **标记价格**：较小灰色字号，带虚线下划线；点击可展示说明（标记价格用于计算盈亏与强平）。

- 两价随行情实时更新。

    

## 2\.6 买单区（Bids）

- 位于盘口下半部分，买方挂单，价格**从高到低**排列。

- 每行：价格（绿色）、数量。

- 每行背景有**绿色深度渐变条**，宽度表示买盘深度占比；自右向左延伸。

    

## 2\.7 盘口精度切换

- 盘口底部左侧提供精度选择按钮，显示当前精度（如 `0.1`）及下拉箭头。

- 点击从底部弹出精度列表弹层：0\.1、1、10、100、1000。

- 选择后盘口价格聚合刷新，列表关闭。

    

## 2\.8 数据更新

- 盘口数据通过 WebSocket 或轮询实时更新；仅刷新变化行，避免整表闪烁。

- 数量大额使用适当缩写（K/M/B）。

    

## 2\.9 新币未开盘状态（盘口区）

当选中尚未正式开启交易的合约时，正常盘口内容隐藏，盘口区域被**待开盘倒计时面板**覆盖：



**（1）展示内容**

- 币种图标与「新幣上線」标签。

- 交易对名称（如 PEPE / USDT）。

- 文案：「即將開啟交易」。

- 倒计时：天 : 时 : 分 : 秒，每秒递减。

    

**（2）联动规则**

- 顶部 24h 涨跌幅徽章显示 `--`，灰色样式。

- 资金费率倒计时显示 `--`。

- 交易终端「最大可開」「強平價格」等字段显示 `--`（见 8\.2）。

- 盘口深度条停止刷新。

# 3\. 交易终端

交易终端位于中部交易区**右侧**，是用户执行下单操作的核心区域。业务规则与 Web 端交易终端一致，交互以 APP 底部弹层、内联选择器为主。



## 3\.1 功能概述

支持开仓/平仓切换、限价/市价/止盈止损下单、保证金模式与杠杆调整、最优价（BBO）、跟随止盈止损、数量单位切换及下单二次确认。



## 3\.2 仓位模式与杠杆

### 3\.2\.1 仓位模式切换

**业务规则（与 Web 一致）：**

- 每个交易对的全仓/逐仓独立设置；首次默认全仓。

- 存在未平仓持仓时**不允许切换**，确认时提示：`当前合约存在持仓，无法切换保证金模式。`

- 底部说明：「调整保证金模式仅对当前合约生效。」

    

**APP 交互：**

- 交易终端顶部左侧按钮显示当前模式（如「全倉」）。

- 点击从底部弹出「保證金模式」弹层：全仓 / 逐仓两个可选卡片，选中高亮；底部「確認」保存并关闭。

    

### 3\.2\.2 杠杆倍数设置

**业务规则（与 Web 一致）：**

- 做多杠杆、做空杠杆**独立**设置，显示为 `20x / 10x`（绿/红分色）。

- 杠杆档位由后台杠杆阶梯配置，非任意输入。

- 有持仓时对应方向杠杆**只允许调高**，调低提示：`当前合约存在持仓，无法调低杠杆。请先平仓后再操作。`

- 可开名义价值随杠杆阶梯联动计算。

    

**APP 交互：**

- 点击杠杆按钮，从底部弹出「調整槓桿」弹层（可滚动，最高约 85% 屏高）。

- 弹层分「做多杠杆」「做空杠杆」两个区域，各含：当前倍数预览、步进器（− / 数值 / \+）、滑块、调整后最大可开提示。

- 底部「確認」保存，并同步刷新：终端杠杆按钮、持仓卡片杠杆标签、委托卡片等。

- 持仓卡片内「槓桿」按钮可快捷打开同一弹层。

    

## 3\.3 开仓/平仓切换

**（1）切换控件**

- 灰底分段控件：「開倉」（默认白底阴影高亮）、「平倉」（灰色）。

- 互斥切换。

    

**（2）联动规则**

- 切换至**平仓**时：隐藏「跟随止盈止损」勾选区域；显示平仓按钮区（可平多/可平空数量 + 平多/平空按钮）。

- 切换至**开仓**时：恢复跟随止盈止损区域；显示开多/开空按钮及最大可开、强平价格提示。

- 订单类型为「止盈止损」时，触发价输入区随开/平仓模式变化（见 3\.5\.3）。

    

**（3）下单按钮（开仓模式）**

- 绿色「買入 / 做多」：开多方向。

- 红色「賣出 / 做空」：开空方向。

- 各自上方展示：最大可開、強平價格（该方向）。

    

**（4）下单按钮（平仓模式）**

- 展示可平多数量 + 红色「平多」按钮。

- 展示可平空数量 + 绿色「平空」按钮；无可平空仓时按钮置灰禁用。

- 点击平多/平空进入下单确认流程（交易终端内）；**持仓卡片「平倉」按钮**则先打开平仓设置弹层（见 4\.3\.3）。

    

## 3\.4 订单类型选择

**（1）触发方式**

- 订单类型行：左侧信息图标 + 当前类型文案（如「限價委託」）+ 右侧箭头。

- 点击从底部弹出「訂單類型」弹层。

    

**（2）可选项**

|选项|说明|
|---|---|
|限價委託|指定价格挂单|
|市價委託|按市场最优价成交|
|止盈止損|条件触发单（独立订单类型）|

- 选中后更新订单类型文案，关闭弹层，并联动价格区、止盈止损区显示/隐藏。

- 弹层底部提供「取消」按钮。

    

## 3\.5 订单类型与下单参数

### 3\.5\.1 限价单

**参数（与 Web 一致）：**

|字段|说明|
|---|---|
|價格|可编辑，默认填入最新价；单位 USDT|
|數量|手动输入 + 百分比滑杆（0%/25%/50%/75%/100%）|
|货币单位|BTC / USDT 切换，切换后清空并按规则换算|
|可用余额|展示可用 USDT；右侧「+」进入充币/划转（见 3\.10）|
|止盈/止损（可选）|跟随止盈止损勾选（仅开仓模式）|

**价格区交互：**

- 价格输入框上方浮层标签「價格」。

- 右侧「最优价」按钮（见 3\.6）：开启后价格区变为 BBO 选项展示，点击打开最优价选择弹层。

    

**数量区交互：**

- 输入框右侧内嵌货币单位选择器，点击弹出「货币单位」底部列表（BTC / USDT）。

- 下方 5 档圆点百分比滑杆（视觉刻度 0/25/50/75/100%），拖动按可用余额比例填充数量。

    

**校验（与 Web 一致）：**

- 价格：数值、\>0、符合价格精度。

- 数量：符合精度、≤ 最大可开、≥ 最小交易单位。

    

### 3\.5\.2 市价单

**参数：**

|字段|说明|
|---|---|
|價格|固定显示「市價」，不可编辑；最优价按钮不可用|
|數量|同限价单|
|止盈/止损（可选）|同限价单|

- 订单有效期后端为 IOC（与 Web 一致）。

    

### 3\.5\.3 止盈止损单

**功能定位（与 Web 5\.4\.3 一致）：**

- 条件订单：预设触发价，达标后自动提交开仓/平仓委托。

- 与限价/市价上的「跟随止盈止损」勾选框是**不同功能**。

    

**（1）委托模式**

- 顶部内联选择器，默认「单向」；点击打开通用选择器弹层：

    - **单向**：预设触发价与委托价；市价达触发价后按委托价下单。

    - **双向**：预设止盈/止损触发价与委托价；一侧触发成交后另一侧自动失效。

    

**（2）输入区结构（动态渲染）**

每个触发条件行包含：

|元素|说明|
|---|---|
|标签|开仓双向：两个「触发价格」；开仓单向：一个「触发价格」；平仓双向：「止盈」「止损」；平仓单向：「触发价格」|
|触发价输入|数字输入框|
|触发价类型|内联选择器，默认「最新价格」；可选「标记价格」（APP 原型）；Web 另支持指数价格|
|委托类型|内联选择器，文案如「市價委託」「限價委託」「止盈市價」等，随开/平仓及双向/单向变化|
|委托价输入|选限价时显示；选市价时隐藏|

- 委托类型、触发价类型均通过**通用选择器弹层**（`picker-sheet`，z-index 70）选择，确保叠置于交易相关弹层之上。

    

**（3）方向与按钮**

- **开仓**：用户点击「買入/做多」或「賣出/做空」提交，方向由按钮决定。

- **平仓**：根据持仓方向显示平多/平空；止盈止损判定规则与 Web 一致（触发价与开仓均价关系）。

    

**（4）验证规则**

**触发价验证：**

|场景|验证规则|
|---|---|
|开仓双向 \+ 开多|两个触发价均须 \> 当前最新成交价|
|开仓双向 \+ 开空|两个触发价均须 \< 当前最新成交价|
|开仓单向 \+ 开多|触发价须 \> 当前最新成交价|
|开仓单向 \+ 开空|触发价须 \< 当前最新成交价|
|平仓双向（持多）|止盈触发价 \> 持仓开仓均价；止损触发价 \< 持仓开仓均价|
|平仓双向（持空）|止盈触发价 \< 持仓开仓均价；止损触发价 \> 持仓开仓均价|
|平仓单向（持多/持空）|触发价须 ≠ 持仓开仓均价|

- 所有触发价不能等于当前最新成交价。

- 触发价小数精度须符合交易对价格精度。

    

**委托价验证（仅限价委托时）：**

|方向|验证规则|
|---|---|
|开多|委托价 ≥ 触发价|
|开空|委托价 ≤ 触发价|
|平多|委托价 ≥ 触发价|
|平空|委托价 ≤ 触发价|

    

**数量验证：**

- 与限价单一致（精度、最大可开、最小交易单位）；开仓须检查保证金充足。

    

**（5）委托类型文案（APP 展示）**

内联选择器根据场景显示不同文案，便于用户识别：

|场景|市价|限价|
|---|---|---|
|开仓|市價委託|限價委託|
|平仓双向·止盈|止盈市價|止盈限價|
|平仓双向·止损|止損市價|止損限價|
|平仓单向|平倉市價|平倉限價|

    

**（6）生命周期**

- 未触发：当前委托 → 止盈止损单子 Tab；可撤单、修改。

- 已触发：历史委托；双向单一边成交后另一边「已取消」。

    

### 3\.5\.5 订单有效期

与 Web 端 5\.6 一致：

|类型|限价单|市价单|止盈止损单|
|---|---|---|---|
|GTC（一直有效）|✓|—|—|
|FOK（全部成交或取消）|✓|—|—|
|IOC（立即成交或取消）|✓|默认|—|
|GTC（直至触发）|—|—|默认|

- APP 若未单独展示有效期选择 UI，由订单类型隐式决定并提交后端。

    

### 3\.5\.4 数量输入（共用）

- 限价、市价、止盈止损共用数量输入框与百分比滑杆。

- 止盈止损双向模式下，两个触发条件共用同一数量。

    

## 3\.6 最优价功能（仅限价单）

**选项（与 Web 一致）：**

|选项|含义|
|---|---|
|对手价 1（默认）|卖一/买一|
|对手价 5|卖五/买五|
|同向价 1|己方最优一档|
|同向价 5|己方最优五档|

**APP 交互：**

- 点击「最优价」进入 BBO 模式：价格输入框切换为选项展示（如「对手价 1」+ 箭头）。

- 再次点击「最优价」或点击价格区，打开「最优价」底部弹层，含各选项说明与勾选标记。

- 选中后回填展示文案；实际限价下单时按盘口实时映射价格。

- 市价单、止盈止损单模式下不可用。

    

## 3\.7 跟随止盈止损（勾选框）

**（与 Web 5\.7 一致）**

- 仅**开仓模式**且订单类型为限价/市价时显示。

- 勾选「止盈/止損」展开：

    - 止盈：触发价输入 + 触发价类型选择（默认最新价格）

    - 止损：触发价输入 + 触发价类型选择

- 标签带虚线下划线；APP 可点击展示说明文案。

- 主订单完全成交后激活；部分成交按比例；主单取消则不激活。

- 验证：做多 TP \> 入场、SL \< 入场；做空相反。

    

## 3\.8 开仓/平仓下单确认

### 3\.8\.1 流程（与 Web 5\.8 一致）

1. 点击下单按钮 → 若偏好未关闭二次确认，弹出居中确认弹窗。

2. 用户点击「確認」→ 提交订单；点击 ✕ 或「取消」→ 放弃。

3. 勾选「不再为{订单类型}提示二次确认」且点击确认后，偏好持久化。

4. 点击遮罩**不关闭**弹窗（防误触）。

    

### 3\.8\.2 确认弹窗字段

**限价/市价开仓：**

|字段|说明|
|---|---|
|标题|如 `BTCUSDT 開多` / `開空`，方向色|
|價格|限价数值或「市價」|
|數量|含单位|
|保證金|开仓占用|
|預估強平價|按规则计算，不完整时 `--`|
|止盈/止損|已设置则显示触发价，否则 `--`|

**平仓确认：**

- 无保证金字段；其余类似。

- 标题如 `BTCUSDT 平多`，确认按钮橙色。

    

**止盈止损确认弹窗（独立订单类型）：**

|场景|标题|方向标签|
|---|---|---|
|单向开仓 \+ 开多/开空|止盈止損確認|多單（绿）/ 空單（红）|
|双向开仓|止盈止損確認|多單 / 空單|
|双向/单向平仓|止盈止損確認|多單 / 空單|

公共字段：合约信息、开仓价格（仅平仓）、触发价格（含类型标注）、委托价格、预估收益（平仓场景）、下单数量。

底部：「不再为止盈止损订单提示二次确认」勾选、「取消」「確認提交」。

    

**持仓止盈止损确认：**

- 由持仓卡片「止盈止損」弹层内「確認設置」触发，字段与 Web 持仓止盈止损确认一致（见 4\.2\.3）。

    

### 3\.8\.3 偏好设置

- 入口：更多菜单 → **偏好设置** → 跳转 `合约偏好设置.html` 子页（非抽屉）。

- 开关项：限价单、市价单、止盈止损单二次确认；成交提示音等（与 Web 1\.1 规则一致）。

- 本地/服务端持久化；支持恢复默认。

    

### 3\.8\.4 结果反馈

- 成功：Toast 提示「下单成功」等。

- 失败：展示后端返回错误文案（如名义价值不足）。

- APP Toast 居中偏上显示，约 1\.6s 自动消失。

    

## 3\.9 交易终端数据摘要

开仓模式下，每个方向按钮上方展示：

|字段|说明|
|---|---|
|最大可開|该方向剩余可开名义价值（USDT）|
|強平價格|按当前参数预估的强平触发价|

- 新币未开盘或数据不足时显示 `--`。

- 计算公式与 Web 端 5\.10 一致。

    

## 3\.10 充币/划转入口

- 「可用」余额右侧「+」图标，点击跳转充值页或弹出充币方式选择（与 Web 5\.9 业务一致）。

- 未登录状态下隐藏「+」入口。

# 8\. 特殊页面状态

## 8\.1 未登录状态

**（1）交易终端**

- 开仓/平仓按钮区隐藏，替换为单一「登錄後交易」按钮，点击引导登录。

- 可用余额、最大可开、强平价格、可平数量等显示 `--`。

- 充币「+」隐藏。

    

**（2）仓位管理**

- 持仓工具栏隐藏；持仓卡片隐藏，显示「暫無持倉」。

- 委托子 Tab 数量显示 0；列表「暫無委託」。

- 一键平仓隐藏。

    

## 8\.2 新币未开盘状态

**（1）顶部**

- 涨跌幅徽章 `--`，灰色。

    

**（2）盘口**

- 显示倒计时覆盖层（见 2\.9）。

- 资金费率倒计时 `--`。

    

**（3）交易终端**

- 最大可開、強平價格（多空）均为 `--`。

- 其他输入控件仍可展示，但下单应被后端拦截或前端禁用（以风控为准）。

    

## 8\.3 数据加载与异常

- 首次进入：关键数值可显示 `--` 或骨架屏。

- 盘口/列表拉取失败：展示错误提示，支持下拉刷新。

- 网络恢复后自动重连并刷新行情与盘口。

# 11\. 附录 B：预估强平价格与保证金展示规则

与 Web 端 5\.10 / 5\.8 一致，APP 在交易终端摘要、下单确认弹窗中展示：

**（1）预估强平价格**

- 依据杠杆、开仓价（或市价）、保证金等计算；公式以对方系统为准。

- 做多且杠杆 ≤ 1 时显示 `--`。

- 价格/数量输入不完整时显示 `--`。

- 新币未开盘状态显示 `--`。

    

**（2）保证金**

- 开仓确认弹窗展示该笔预计占用保证金（USDT）。

- 平仓确认不展示保证金字段。

    

**（3）可平数量**

- 平仓模式展示「可平多」「可平空」，随持仓实时更新；无可平空仓时平空按钮禁用。

    

**（4）费率提示**

- 若产品需要展示吃单/挂单费率，可在交易终端底部以小字展示（与 Web 5\.10 费率行一致）。
