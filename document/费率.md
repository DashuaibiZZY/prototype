# VIP 费率与后台配置

# 0. 文档概述

本文档描述 ForX 永续合约 **VIP 费率** 的业务逻辑、用户端展示（APP / Web）及后台管理能力，对应原型如下：

| 端 | 原型路径 |
|---|---|
| APP | `perp_dex/app/VIP费率等级.html` |
| Web | `perp_dex/投资组合.html`（区块二：VIP 权益卡片） |
| 后台 · 费率 | `perp_dex/admin/用户费率设置.html` |
| 后台 · 费率日志 | `perp_dex/admin/费率操作记录.html` |

> 用户风控冻结能力见 `document/用户风控冻结.md`，两者权限隔离（费率管理员 / 风控管理员）。

# 一、VIP 费率业务逻辑

## 1.1 VIP 自然升降级

### 统计口径

- **统计周期**：近 **14 日**合约交易量（USD）。
- **结算时点**：每天 **UTC+8 23:59** 取数，按过去 14 日累计交易量匹配 VIP 等级。
- **生效规则**：
  - 交易量**达标**对应档位后，按匹配等级生效费率。
  - 交易量**不达标**时**立即滑档**（降回实际交易量对应的 VIP 等级），不保留历史最高等级。

### VIP 等级与费率表

| VIP 等级 | 14 天交易量门槛（USD） | Maker 费率 | Taker 费率 |
|---|---|---|---|
| VIP 0 | < $5M | 0.015% | 0.045% |
| VIP 1 | ≥ $5M 且 < $25M | 0.013% | 0.039% |
| VIP 2 | ≥ $25M 且 < $100M | 0.010% | 0.034% |
| VIP 3 | ≥ $100M 且 < $500M | 0.006% | 0.029% |
| VIP 4 | ≥ $500M | 0.004% | 0.026% |

### 用户端展示说明

- 用户当前生效的 Maker / Taker 费率，以**实际结算后的等级**为准。
- 升级进度条、距下一等级差额等，均基于近 14 日交易量与上表门槛计算。
- 页面 FAQ 可做简化表述；**以本文档 1.1 节结算规则为准**（每日 UTC+8 23:59 取数、不达标立即滑档）。

---

## 1.2 运营后台配置费率

运营可在后台对用户进行**人工费率干预**，与 VIP 自然升降级并行存在，但优先级更高（见 1.3）。

### 配置方式

| 方式 | 说明 |
|---|---|
| **指定 VIP 等级** | 手动套用 VIP 1–VIP 4 的标准费率（**不含 VIP 0**，VIP 0 仅由交易量自然判定） |
| **自定义费率** | 单独设置 Taker / Maker 费率（列表展示为「自定义」，不显示 VIP 0） |

### 生效规则

- 后台配置保存后**立即生效**。
- 一旦被运营配置费率，该用户**不再参与 VIP 自然升降级**（冻结在自然升降通道之外，按运营配置费率计费）。
- **撤销运营配置**后，用户恢复按近 14 日交易量匹配 VIP 等级的自然费率。

### 保存必填材料

配置或变更费率时，须上传：

1. **审批截图**（内部审批流程凭证）
2. **用户价值证明**（其他交易所交易量 / VIP 等级等截图）
3. **操作备注**（变更原因）

### 撤销费率设置

- 入口：用户费率设置列表 / 配置抽屉内「撤销费率设置」。
- 条件：仅对当前存在**运营配置**的用户展示。
- 效果：清除人工费率，恢复 VIP 自然升降级逻辑；下一笔交易起按自然等级费率计费。
- 须填写操作备注；建议同样留存审批依据（原型阶段可与配置共用备注校验）。

---

## 1.3 费率优先级

同一用户、同一笔合约交易，按以下优先级取费率（高 → 低）：

```
交易对独立配置费率（默认不存在）
        ↓
用户运营配置费率（VIP 等级 / 自定义）
        ↓
VIP 自然等级匹配费率（近 14 日交易量）
```

说明：

- **交易对独立费率**：预留能力，默认全站不启用；若某交易对单独配置，则覆盖用户级与 VIP 级费率。
- **运营配置**：后台人工设置，生效期间屏蔽自然升降级。
- **自然等级**：无运营配置时的默认路径。

---

## 1.4 费率操作审计

费率配置与撤销均为敏感操作，所有写操作须记录审计日志：

| 模块 | 日志页面 |
|---|---|
| 费率配置 / 撤销 | `费率操作记录.html` |

每条日志至少包含：**操作人、操作时间、目标用户（UID / 钱包）、操作类型、变更详情、备注**；另关联审批材料存档。

# 二、APP 与 Web 页面

## 2.1 APP · VIP 费率等级页

**原型**：`perp_dex/app/VIP费率等级.html`  
**入口**：首页快捷入口「VIP」、奖励中心 VIP 卡片、合约交易页等。

### 页面结构

| 区块 | 内容 |
|---|---|
| 顶部导航 | 标题「VIP 费率等级」+ 返回 |
| 当前 VIP 信息卡 | 当前等级、近 14 日交易量、Maker/Taker 费率、升级进度、去交易按钮 |
| VIP 费率等级表 | 横向滚动表格，当前等级行高亮 |
| VIP 专属权益 | 资讯研报、礼物、会场、星际徽章（展示型）等 |
| 常见问题 | 折叠 FAQ |

### 当前 VIP 信息卡

| 字段 | 说明 |
|---|---|
| 当前 VIP 等级 | 如 VIP 2 胶囊标签 |
| 当前 14 天交易量 | USD 金额 |
| Maker / Taker 费率 | 当前生效费率 |
| 升级进度 | 进度条 + 百分比 + 距下一等级差额与目标门槛 |
| 去交易 | 跳转合约交易页 |

**展示费率**：为用户**当前实际生效**费率（已综合 1.3 优先级；若存在运营配置，展示运营费率，进度条仍可按自然交易量展示参考）。

### VIP 费率等级表

- 列：VIP 等级、14 天交易量门槛、Maker 费率、Taker 费率。
- 当前等级行高亮（`current-row`）。

### 交互

- FAQ 项点击展开 / 收起。
- 「去交易」→ 合约交易页。
- 等级表支持横向滑动（小屏）。

# 三、后台管理页面

## 3.1 用户费率设置

**原型**：`perp_dex/admin/用户费率设置.html`  
**权限**：费率管理员。

### 列表筛选

| 字段 | 说明 |
|---|---|
| 钱包地址 | 模糊匹配 |
| UID | 模糊匹配 |

### 列表字段

| 列 | 说明 |
|---|---|
| UID / 钱包 | 用户标识 |
| 配置等级 | 运营配置：VIP 1–4 或「自定义」；未配置时显示「自然升降」 |
| 交易量对应等级 | 按近 14 日交易额自动计算的 VIP 等级 |
| 费率 (Taker / Maker) | 当前生效费率 |
| 近 30 日交易额 | 统计展示 |
| 近 30 日手续费收入 | 平台从该用户收取的手续费 |
| 近 14 日交易额 | 自然升降级统计口径 |
| 近 14 日手续费收入 | 统计展示 |
| 操作 | 配置费率 / 撤销费率设置 |

### 配置抽屉

1. **VIP 费率参考表**（VIP 0–4 全量门槛与费率）
2. **费率配置**：VIP 1–4 或自定义 Taker / Maker
3. **审批材料**（必填）：审批截图、用户价值证明截图
4. **操作备注**（必填）

### 撤销费率设置

- 对存在运营配置的用户，操作列额外展示「撤销费率设置」。
- 确认后清除人工配置，恢复自然升降级；写入费率操作记录。

### 子页面

- 侧栏入口 → **费率操作记录**（`费率操作记录.html`）。

# 附录：名词说明

| 名词 | 说明 |
|---|---|
| 自然升降 | 按近 14 日交易量每日 UTC+8 23:59 结算的 VIP 等级机制 |
| 运营配置 | 后台人工指定的 VIP 等级或自定义费率 |
| 交易量对应等级 | 仅按 14 日交易量计算的理论 VIP 等级，不代表一定为当前生效等级 |
| 配置等级 | 运营是否介入及介入方式（自然升降 / VIP n / 自定义） |
